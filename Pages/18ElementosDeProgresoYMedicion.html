CURSO DE INICIACIÓN A HTML

Clase 18: Elementos de Progreso y Medición.

Además de los campos de entrada para recopilar información, HTML5 nos ofrece elementos semánticos para visualizar el progreso de una tarea o la magnitud de un valor dentro de un rango definido. Estos elementos, <progress> y <meter>, mejoran la comunicación con el usuario de manera visual y accesible.

18.1. <progress> para barras de progreso.

El elemento <progress> se utiliza para visualizar el progreso de una tarea. Es ideal para indicar cuánto se ha completado de un proceso que se espera que finalice.

Atributos principales:

  • value: Indica la cantidad de trabajo ya completado. Debe ser un número de punto flotante válido entre 0 y el valor de max.
  • max: Define el valor que representa el 100% de la tarea. Si se omite, el valor por defecto es 1.

Si el atributo value no está presente, la barra de progreso se muestra como una barra indeterminada (animada), lo que indica que una tarea está en curso, pero el progreso total es desconocido (ej., "cargando...").

Ejemplo: Descarga de un archivo.

<form>
    <label for="descarga">Progreso de la descarga:</label>
    <progress id="descarga" value="70" max="100">70%</progress>
    <p>Archivo descargado: 70%</p>

    <label for="carga_indeterminada">Cargando contenido...</label>
    <progress id="carga_indeterminada">Cargando...</progress>
</form>
70%

Archivo descargado: 70%

Cargando...

Contenido interno: El texto o HTML dentro de las etiquetas <progress>...</progress> (como "70%") se muestra en navegadores que no soportan el elemento <progress> (o por parte de lectores de pantalla), actuando como un fallback.

18.2. <meter> para mediciones.

El elemento <meter> se utiliza para mostrar la medición escalar dentro de un rango conocido. Es adecuado para indicar el nivel actual de algo (ej., uso del disco, nivel de batería, relevancia de un resultado).

Atributos principales:

  • value: El valor medido actual.
  • min: El valor más bajo del rango. Por defecto es 0.
  • max: El valor más alto del rango. Por defecto es 1.
  • low: El valor que define el límite inferior de la zona "baja". Si value está por debajo de low, se considera "bajo".
  • high: El valor que define el límite superior de la zona "alta". Si value está por encima de high, se considera "alto".
  • optimum: El valor ideal o "óptimo" dentro del rango. Ayuda a los navegadores a determinar si un valor "bajo" o "alto" es bueno o malo (ej., bajo nivel de batería es malo, baja latencia es buena).

Ejemplo: Uso del disco y nivel de batería.

<form>
    <label for="uso_disco">Uso del disco (GB):</label>
    <meter id="uso_disco" value="70" min="0" max="100" low="20" high="80" optimum="50">70 GB usados de 100 GB</meter>
    <p>Espacio: 70/100 GB (Bajo < 20, Alto > 80, Óptimo en 50)</p>

    <label for="nivel_bateria">Nivel de batería:</label>
    <meter id="nivel_bateria" value="0.25" min="0" max="1" low="0.1" high="0.7" optimum="0.9">25%</meter>
    <p>Batería: 25% (Bajo < 10%, Alto > 70%, Óptimo en 90%)</p>
</form>
70 GB usados de 100 GB

Espacio: 70/100 GB (Bajo < 20, Alto > 80, Óptimo en 50)

25%

Batería: 25% (Bajo < 10%, Alto > 70%, Óptimo en 90%)

Diferencia con <progress>: Mientras <progress> indica una tarea que avanza hacia una finalización, <meter> representa una medida estática o un valor actual dentro de un rango establecido.

18.3. Casos de uso para estos elementos.

Ambos elementos son valiosos para proporcionar información clara y concisa a los usuarios:

  • <progress>:
    • Barras de carga de páginas, imágenes o videos.
    • Estado de envío de formularios complejos.
    • Progreso de un asistente paso a paso.
    • Indicadores de instalación de software.
  • <meter>:
    • Indicadores de uso de recursos (CPU, memoria, disco).
    • Niveles de batería de un dispositivo.
    • Relevancia de resultados de búsqueda.
    • Puntuaciones en un juego (ej., nivel de salud, mana).
    • Calificaciones de productos o servicios (ej., promedio de estrellas).
    • Cantidad de elementos en un carrito de compras.

Estilo CSS: Ambos elementos <progress> y <meter> pueden ser estilizados con CSS para adaptarse al diseño de tu sitio web, aunque la personalización de su apariencia interna (como el color de la barra en diferentes estados de <meter>) puede ser más compleja y variar entre navegadores.
