Clase 13: Otros Tipos de Entrada I: Selección.

Además de los campos de texto, los formularios HTML ofrecen una variedad de elementos para que los usuarios elijan entre opciones predefinidas. Estos elementos son cruciales para guiar al usuario, estandarizar las respuestas y mejorar la usabilidad. En esta clase, exploraremos los botones de opción (radio buttons), las casillas de verificación (checkboxes) y las listas desplegables, junto con la importante etiqueta <label> que asocia texto descriptivo a los controles de formulario.

13.1. checkbox y radio (grupos de radio buttons).

Estos dos tipos de <input> permiten al usuario seleccionar opciones de una lista.

13.1.1. <input type="checkbox">

Permite al usuario seleccionar cero, una o varias opciones de un conjunto.

  • Cada checkbox es independiente, y se envía su valor si está marcado.
  • Se utiliza para opciones que no son mutuamente excluyentes (ej., "Deseo recibir boletines", "Acepto los términos y condiciones", "Selecciona tus intereses").

Ejemplo:

<form>
    <p>Selecciona tus intereses:</p>
    <div class="checkbox-group">
        <input type="checkbox" id="interes1" name="intereses" value="html">
        <label for="interes1">HTML</label><br>
        
        <input type="checkbox" id="interes2" name="intereses" value="css">
        <label for="interes2">CSS</label><br>
        
        <input type="checkbox" id="interes3" name="intereses" value="js">
        <label for="interes3">JavaScript</label>
    </div>
</form>

13.1.2. <input type="radio"> (grupos de radio buttons)

Permite al usuario seleccionar una y solo una opción de un conjunto. Las opciones son mutuamente excluyentes.

  • Para agrupar botones de radio, deben compartir el mismo atributo name. Esto asegura que solo se pueda seleccionar uno dentro de ese grupo.
  • Se utiliza para opciones donde solo una respuesta es válida (ej., "Sexo", "Método de pago", "Nivel de experiencia").

Ejemplo:

<form>
    <p>Selecciona tu sexo:</p>
    <div class="radio-group">
        <input type="radio" id="masculino" name="sexo" value="masculino">
        <label for="masculino">Masculino</label><br>
        
        <input type="radio" id="femenino" name="sexo" value="femenino">
        <label for="femenino">Femenino</label><br>
        
        <input type="radio" id="otro" name="sexo" value="otro">
        <label for="otro">Otro</label>
    </div>
</form>

Atributo checked: Puedes usar el atributo checked en un checkbox o radio button para que aparezca seleccionado por defecto al cargar la página: <input type="radio" checked>.

13.2. La etiqueta <label> y su asociación con los inputs.

La etiqueta <label> es fundamental para la accesibilidad y la usabilidad de los formularios. Sirve para proporcionar una descripción textual (una "etiqueta") para un control de formulario.

  • Asociación explícita: La mejor práctica es asociar un <label> con un control de formulario usando el atributo for del <label>, cuyo valor debe coincidir con el id del control del formulario.
  • Beneficios:
    • Cuando el usuario hace clic en el texto de la etiqueta, el control de formulario asociado recibe el foco o se activa (ej., un checkbox se marca).
    • Los lectores de pantalla leen la etiqueta junto con el tipo de control, lo que es vital para la comprensión del formulario por parte de usuarios con discapacidad visual.

Ejemplo de asociación con for y id:

<label for="email_contacto">Correo Electrónico:</label>
<input type="email" id="email_contacto" name="email">

Asociación implícita (menos común): También se puede envolver el input dentro de la etiqueta <label>, pero la asociación explícita (for/id) es generalmente preferida por su claridad y flexibilidad en el styling.

13.3. Listas desplegables: <select>, <option>, <optgroup>.

Las listas desplegables (o menús desplegables) permiten a los usuarios seleccionar una o varias opciones de una lista predefinida.

13.3.1. <select>

Es la etiqueta contenedora para la lista desplegable. Su atributo name es el que se envía al servidor con el valor de la opción seleccionada.

  • size: (Opcional) Número de opciones visibles sin necesidad de desplazamiento. Si es mayor que 1, la lista se muestra como una caja de desplazamiento en lugar de un menú desplegable.
  • multiple: (Opcional) Si está presente, permite al usuario seleccionar múltiples opciones (normalmente manteniendo Ctrl/Cmd mientras hace clic). Los valores seleccionados se envían como un array al servidor.

Ejemplo:

<form>
    <label for="paises">Selecciona un país:</label>
    <select id="paises" name="pais">
        <option value="es">España</option>
        <option value="mx">México</option>
        <option value="ar">Argentina</option>
    </select>
</form>

13.3.2. <option>

Representa una de las opciones individuales dentro de un <select>.

  • value: El valor que se envía al servidor cuando esta opción es seleccionada. Si se omite, se envía el texto visible de la opción.
  • selected: (Opcional) Si está presente, esta opción aparecerá seleccionada por defecto al cargar la página.

Ejemplo con value y selected:

<select name="color">
    <option value="red">Rojo</option>
    <option value="blue" selected>Azul</option>
    <option value="green">Verde</option>
</select>

13.3.3. <optgroup>

Se utiliza para agrupar opciones relacionadas dentro de un <select>, creando una estructura jerárquica. La etiqueta label de <optgroup> se muestra como un encabezado para el grupo.

Ejemplo:

<select name="fruta">
    <optgroup label="Cítricos">
        <option value="naranja">Naranja</option>
        <option value="limon">Limón</option>
    </optgroup>
    <optgroup label="Frutas con hueso">
        <option value="melocoton">Melocotón</option>
        <option value="ciruela">Ciruela</option>
    </optgroup>
</select>

Usabilidad: Los menús desplegables son ideales para listas largas de opciones. Para listas cortas (2-5 opciones), los botones de radio suelen ser más intuitivos.
