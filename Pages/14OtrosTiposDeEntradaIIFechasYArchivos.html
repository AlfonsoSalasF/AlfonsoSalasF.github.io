Clase 14: Otros Tipos de Entrada II: Fechas y Archivos.

Continuando con los elementos de entrada de formularios, en esta clase exploraremos tipos de <input> que permiten a los usuarios seleccionar fechas, horas y cargar archivos. Estos tipos ofrecen interfaces de usuario especializadas y mejoran la experiencia, además de proporcionar validación básica directamente desde el navegador.

14.1. Tipos de entrada de fecha y hora: date, time, datetime-local, month, week.

HTML5 introdujo varios tipos de <input> específicos para la selección de fechas y horas, que a menudo proporcionan interfaces de calendario o reloj nativas del navegador, lo que facilita enormemente la entrada de datos para el usuario.

14.1.1. <input type="date">

Permite al usuario seleccionar una fecha (año, mes, día). El navegador generalmente muestra un selector de calendario.

Uso común: Fechas de nacimiento, fechas de eventos, fechas de inicio/fin.

Ejemplo:

<form>
    <label for="fecha_evento">Fecha del evento:</label>
    <input type="date" id="fecha_evento" name="fecha_evento">
</form>

14.1.2. <input type="time">

Permite al usuario seleccionar una hora (horas y minutos, opcionalmente segundos y fracciones de segundo).

Uso común: Horas de citas, horarios de apertura/cierre.

Ejemplo:

<form>
    <label for="hora_cita">Hora de la cita:</label>
    <input type="time" id="hora_cita" name="hora_cita">
</form>

14.1.3. <input type="datetime-local">

Combina la selección de fecha y hora en un solo campo, sin incluir información de zona horaria.

Uso común: Fechas y horas de reuniones locales, horarios de reservas.

Ejemplo:

<form>
    <label for="cita_completa">Fecha y hora de la reunión:</label>
    <input type="datetime-local" id="cita_completa" name="cita_completa">
</form>

14.1.4. <input type="month">

Permite al usuario seleccionar un mes y un año.

Uso común: Fechas de caducidad de tarjetas de crédito (solo mes/año), períodos fiscales.

Ejemplo:

<form>
    <label for="fecha_caducidad">Mes y año de caducidad (MM/AAAA):</label>
    <input type="month" id="fecha_caducidad" name="fecha_caducidad">
</form>

14.1.5. <input type="week">

Permite al usuario seleccionar una semana y un año.

Uso común: Planificación de proyectos por semanas, informes semanales.

Ejemplo:

<form>
    <label for="semana_proyecto">Semana del proyecto:</label>
    <input type="week" id="semana_proyecto" name="semana_proyecto">
</form>

14.2. Entrada de archivos: <input type="file"> para subir archivos.

El tipo file permite a los usuarios seleccionar uno o varios archivos de su dispositivo para subirlos al servidor. Es crucial para funcionalidades como la subida de imágenes de perfil, documentos, etc.

Importante: Para que la subida de archivos funcione correctamente, el formulario debe tener el atributo enctype="multipart/form-data" en la etiqueta <form>, y el método debe ser POST.

Atributos comunes:

  • accept: Especifica los tipos de archivo que el servidor aceptará, o que el dispositivo de entrada debería aceptar (ej., image/* para cualquier imagen, .pdf para PDFs).
  • multiple: Si está presente, permite al usuario seleccionar varios archivos a la vez.

Ejemplo:

<form action="/upload_file.php" method="POST" enctype="multipart/form-data">
    <label for="subir_imagen">Sube tu imagen de perfil (JPG/PNG):</label>
    <input type="file" id="subir_imagen" name="imagen_perfil" accept="image/jpeg, image/png">
    <br><br>
    <label for="subir_documentos">Sube documentos (múltiples PDF/DOCX):</label>
    <input type="file" id="subir_documentos" name="documentos[]" accept=".pdf,.docx" multiple>
    <br><br>
    <button type="submit">Subir</button>
</form>




Nombre del campo para múltiples archivos: Cuando se usa multiple, es común añadir [] al final del atributo name (ej., name="documentos[]") para que el servidor pueda interpretar los archivos como un array.

14.3. Atributos min, max, step para entradas numéricas y de fecha/hora.

Estos atributos, vistos previamente con type="number", también son aplicables a los tipos de entrada de fecha y hora para definir rangos y granularidad.

  • min: Establece el valor mínimo permitido.
  • max: Establece el valor máximo permitido.
  • step: Define los incrementos válidos para el valor (ej., cada 5 minutos, cada 7 días).

Ejemplo con date:

<form>
    <label for="fecha_reservacion">Fecha de reservación (solo en 2025):</label>
    <input type="date" id="fecha_reservacion" name="fecha_reservacion" min="2025-01-01" max="2025-12-31">
</form>

Ejemplo con time (pasos de 15 minutos):

<form>
    <label for="hora_reunion">Hora de la reunión (incrementos de 15 min):</label>
    <input type="time" id="hora_reunion" name="hora_reunion" step="900"> <!-- 900 segundos = 15 minutos -->
</form>

Valores de step para tiempo: Para los tipos de tiempo, step se mide en segundos. Por ejemplo, 60 para minutos, 900 para 15 minutos.
