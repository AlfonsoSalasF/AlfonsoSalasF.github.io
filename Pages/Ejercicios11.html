Clase 11: Ejercicios de Conceptos Básicos de Formularios HTML

Ejercicios Prácticos de la Clase 11

Para consolidar tus conocimientos sobre los conceptos básicos de formularios HTML, te propongo los siguientes ejercicios. Intenta resolverlos por ti mismo antes de ver la solución.

Ejercicio 1: Formulario Básico de Contacto

Crea un formulario HTML básico para una sección de "Contacto".

  • El formulario debe tener la etiqueta <form>.
  • Define el atributo action para que los datos se envíen a una URL ficticia como /procesar_contacto.php.
  • Define el atributo method para que los datos se envíen de forma segura (invisible en la URL).
  • Incluye un campo de texto simple para el "Nombre" y un botón para "Enviar". No te preocupes aún por los detalles de los inputs, solo la estructura del formulario.
<form action="/procesar_contacto.php" method="POST">
    <p>Nombre:</p>
    <input type="text" name="nombre">
    <br><br>
    <button type="submit">Enviar</button>
</form>

Ejercicio 2: Formulario de Búsqueda

Crea un formulario simple para una barra de búsqueda en un sitio web.

  • El formulario debe apuntar a la URL /buscar.html.
  • Utiliza el método HTTP adecuado para una búsqueda (donde los parámetros pueden ser visibles en la URL).
  • Debe contener un campo de texto para la consulta de búsqueda y un botón para "Buscar".
<form action="/buscar.html" method="GET">
    <p>Término de búsqueda:</p>
    <input type="text" name="q">
    <br><br>
    <button type="submit">Buscar</button>
</form>

Ejercicio 3: ¿GET o POST?

Para cada uno de los siguientes escenarios, indica si el método GET o POST sería más apropiado y explica brevemente por qué:

  1. Formulario de inicio de sesión (usuario y contraseña).
  2. Envío de un mensaje largo en un foro.
  3. Filtrar productos en una tienda online por categoría y precio.
  4. Formulario para subir una imagen.

Respuestas:

  1. Formulario de inicio de sesión: POST.

    Razón: Se están enviando datos sensibles (contraseña) que no deben ser visibles en la URL. Además, la acción de "iniciar sesión" modifica el estado del usuario en el servidor.

  2. Envío de un mensaje largo en un foro: POST.

    Razón: Se envía un volumen de datos potencialmente grande, y el método GET tiene limitaciones de tamaño en la URL. Además, se está creando o modificando contenido en el servidor.

  3. Filtrar productos en una tienda online: GET.

    Razón: Los parámetros de filtro (categoría, precio) no son sensibles y pueden ser parte de la URL. Esto permite al usuario guardar la URL en favoritos o compartirla para mostrar los resultados con los mismos filtros.

  4. Formulario para subir una imagen: POST.

    Razón: Se está enviando una gran cantidad de datos binarios (la imagen) que no cabrían en una URL. La subida de archivos siempre requiere POST.

Ejercicio 4: Importancia de los Formularios (Reflexión)

En tus propias palabras, explica por qué los formularios HTML son fundamentales para la interacción del usuario en la web. Menciona al menos dos razones clave.

Los formularios HTML son esenciales para la interacción del usuario en la web por varias razones, entre ellas:

  • Recopilación y envío de información: Permiten a los usuarios introducir y enviar datos (texto, selecciones, archivos, etc.) a un servidor. Sin ellos, las páginas web serían estáticas y unidireccionales, incapaces de recoger entradas de los usuarios para acciones como registros, inicios de sesión, comentarios, pedidos de productos, búsquedas, etc.
  • Habilitar la funcionalidad interactiva: Son el mecanismo principal para que los usuarios controlen o modifiquen el contenido o el comportamiento de una aplicación web. Prácticamente cualquier acción que implique una entrada del usuario y una respuesta dinámica (comprar un producto, buscar información, enviar un email desde una web, etc.) se basa en un formulario.

Ejercicio 5: Estructura del Formulario

Crea un formulario para registrar un nuevo usuario. De momento, solo la estructura general.

  • La etiqueta <form> debe tener un action a /registrar_usuario.php y un method apropiado.
  • Dentro del formulario, utiliza elementos como <p> para indicar "Nombre de usuario:", "Email:" y "Contraseña:".
  • Añade inputs de tipo text para el nombre de usuario y email para el correo, y password para la contraseña. No te preocupes por sus atributos name o id aún.
  • Termina con un botón de submit que diga "Registrarme".
<form action="/registrar_usuario.php" method="POST">
    <p>Nombre de usuario:</p>
    <input type="text">
    <br><br>
    
    <p>Email:</p>
    <input type="email">
    <br><br>
    
    <p>Contraseña:</p>
    <input type="password">
    <br><br>
    
    <button type="submit">Registrarme</button>
</form>
