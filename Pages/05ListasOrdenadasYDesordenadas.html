Clase 5: Tipos de Listas en HTML.

Las listas son elementos fundamentales para organizar y presentar información de manera clara y estructurada en la web. HTML nos ofrece diferentes tipos de listas para adaptarse a diversas necesidades, mejorando la legibilidad y la semántica de tu contenido.

5.1. Listas Desordenadas <ul> y <li>

Las listas desordenadas, representadas por la etiqueta <ul> (de "unordered list"), se utilizan para agrupar elementos que no tienen un orden secuencial o específico. Esto significa que el orden en que aparecen los elementos no altera el significado o la información que se transmite. Cada elemento individual dentro de la lista se define con la etiqueta <li> (de "list item"), que por defecto se mostrará con un marcador, usualmente un punto o círculo, aunque este estilo puede ser personalizado completamente con CSS.

Son ideales para:

  • Menús de navegación donde el orden de los ítems no es crucial.
  • Listas de características de un producto o servicio.
  • Enumerar elementos sin una jerarquía temporal o lógica estricta.
Ejemplo de Lista Desordenada: 
                    
                        <ul>
                            <li>Manzanas</li>
                            <li>Plátanos</li>
                            <li>Naranjas</li>
                        </ul>
Resultado:
  • Manzanas
  • Plátanos
  • Naranjas

5.2. Listas Ordenadas <ol> y <li>

Cuando la secuencia o el orden de los elementos es fundamental, recurrimos a las listas ordenadas, que se definen con la etiqueta <ol> (de "ordered list"). Cada elemento sigue siendo una etiqueta <li>. Por defecto, los navegadores numeran estos elementos con dígitos arábigos (1, 2, 3...), pero HTML proporciona atributos para personalizar este comportamiento.

Son perfectas para:

  • Instrucciones paso a paso.
  • Recetas o procedimientos donde el orden es vital.
  • Clasificaciones o rankings.

5.2.1. Atributos Clave de <ol>:

start
Este atributo opcional permite especificar el valor numérico con el que debe comenzar la lista. Es especialmente útil si necesitas continuar una numeración que se inició en otro lugar o si quieres empezar desde un número diferente a 1.
type
Define el tipo de marcador que se usará para los elementos de la lista. Ofrece varias opciones para la numeración:
  • 1: Números arábigos (1, 2, 3, ...) - es el valor por defecto.
  • a: Letras minúsculas (a, b, c, ...)
  • A: Letras mayúsculas (A, B, C, ...)
  • i: Números romanos en minúscula (i, ii, iii, ...)
  • I: Números romanos en mayúscula (I, II, III, ...)
reversed
Este atributo booleano (no requiere valor) hace que la lista se numere en orden descendente, desde el valor inicial hasta 1 (o lo que corresponda si se usa 'start').

5.2.2. Ejemplos de Listas Ordenadas:

a. Lista Ordenada Básica:


                    <ol>
                        <li>Paso uno: Abrir la aplicación.</li>
                        <li>Paso dos: Iniciar sesión con tus credenciales.</li>
                        <li>Paso tres: Navegar al menú principal.</li>
                    </ol>
  1. Paso uno: Abrir la aplicación.
  2. Paso dos: Iniciar sesión con tus credenciales.
  3. Paso tres: Navegar al menú principal.

b. Lista Ordenada con start y type:


                        <ol type="A" start="3">
                            <li>Tercer punto de la secuencia.</li>
                            <li>Cuarto punto a considerar.</li>
                            <li>Quinto y último punto.</li>
                        </ol>
  1. Tercer punto de la secuencia.
  2. Cuarto punto a considerar.
  3. Quinto y último punto.

c. Lista Ordenada con reversed:


                    <ol reversed>
                        <li>Último lugar en la carrera.</li>
                        <li>Penúltimo lugar.</li>
                        <li>Tercer lugar.</li>
                    </ol>
  1. Último lugar en la carrera.
  2. Penúltimo lugar.
  3. Tercer lugar.

5.3. Listas de Definición <dl>, <dt>, <dd>

Las listas de definición son ideales para presentar pares de término-descripción, como un glosario, un diccionario, o una lista de preguntas frecuentes con sus respuestas. A diferencia de las listas ordenadas y desordenadas, no utilizan marcadores ni números por defecto, sino que estructuran el contenido visualmente por indentación.

Este tipo de lista se construye con tres etiquetas principales:

  • <dl> (de "description list" o "definition list"): Es el contenedor principal de toda la lista de definición.
  • <dt> (de "description term" o "definition term"): Define un término que va a ser descrito. Puede haber uno o más términos por cada descripción.
  • <dd> (de "description details" o "definition description"): Proporciona la descripción o definición del término(s) inmediatamente anterior(es).

Las listas de definición ofrecen una excelente estructura semántica para contenido que implica una relación directa entre un concepto y su explicación.

Ejemplo de Lista de Definición: 

<dl>
<dt>HTML</dt>
<dd>Lenguaje de Marcado de Hipertexto, un lenguaje estándar para la creación de páginas web.</dd>

<dt>CSS</dt>
<dd>Hojas de Estilo en Cascada, un lenguaje de hojas de estilo usado para describir la presentación de un documento escrito en HTML.</dd>

<dt>JavaScript</dt>
<dd>Lenguaje de programación ligero, interpretado o compilado 'just-in-time' con funciones de primera clase. Es el lenguaje de programación de la web.</dd>
</dl>
Resultado:
HTML
Lenguaje de Marcado de Hipertexto, un lenguaje estándar para la creación de páginas web.
CSS
Hojas de Estilo en Cascada, un lenguaje de hojas de estilo usado para describir la presentación de un documento escrito en HTML.
JavaScript
Lenguaje de programación ligero, interpretado o compilado 'just-in-time' con funciones de primera clase. Es el lenguaje de programación de la web.
