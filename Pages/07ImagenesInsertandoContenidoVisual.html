CURSO DE INICIACIÓN A HTML

Clase 7: Imágenes: Insertando Contenido Visual.

En el mundo digital actual, las imágenes son un componente esencial de cualquier página web atractiva y efectiva. No solo rompen la monotonía del texto, sino que también pueden transmitir información de manera rápida y visual, mejorar la estética y la experiencia del usuario. En esta clase, aprenderemos cómo insertar imágenes en nuestros documentos HTML utilizando la etiqueta fundamental y sus atributos clave para asegurar que se muestren correctamente y sean accesibles para todos.

7.1. La etiqueta <img> y los atributos src, alt, width, height.

La etiqueta <img> se utiliza para incrustar una imagen en un documento HTML. Es una etiqueta vacía (void element), lo que significa que no tiene una etiqueta de cierre y no puede contener contenido entre sus etiquetas de apertura y cierre.

Los atributos más importantes para la etiqueta <img> son:

7.1.1. Atributo src (Source)

El atributo src (del inglés "source", fuente) es obligatorio y especifica la ruta o URL de la imagen que se va a mostrar. Puede ser una ruta relativa (dentro de tu propio proyecto) o una URL absoluta (una imagen en otro servidor).

Ejemplo:

<img src="imagenes/mi_imagen.jpg">
<img src="https://ejemplo.com/imagen_externa.png">

7.1.2. Atributo alt (Alternative Text)

El atributo alt (del inglés "alternative text", texto alternativo) es también obligatorio y crucial para la accesibilidad y la SEO. Proporciona una descripción textual de la imagen. Este texto se muestra si la imagen no se carga (por un error en la ruta, problemas de conexión, etc.) o es leído por lectores de pantalla para personas con discapacidad visual. También es utilizado por los motores de búsqueda para entender el contenido de la imagen.

Ejemplo:

<img src="imagenes/puesta_de_sol.jpg" alt="Una hermosa puesta de sol sobre el océano con tonos anaranjados y morados.">

Importancia: Nunca dejes el atributo alt vacío a menos que la imagen sea puramente decorativa y no aporte información al contenido (en cuyo caso se usaría alt="", pero con cuidado). Un buen texto alternativo es descriptivo y conciso.

7.1.3. Atributos width y height

Estos atributos opcionales especifican el ancho y el alto de la imagen en píxeles. Aunque puedes usar CSS para controlar el tamaño de las imágenes, especificar width y height directamente en la etiqueta <img> es una buena práctica porque ayuda al navegador a reservar el espacio necesario para la imagen antes de que se cargue, evitando saltos de diseño (layout shifts) que pueden afectar la experiencia del usuario y la puntuación de rendimiento (Core Web Vitals).

Si solo especificas uno (por ejemplo, width), el navegador escalará la imagen proporcionalmente para mantener su relación de aspecto.

Ejemplo:

<img src="../Images/MiniyoC.png" alt="Chico joven estudiante" width="30" height="40">

Si la imagen es muy grande y no se especifica el tamaño, podría sobrepasar los límites del diseño y requerir más tiempo de carga para el navegador para determinar sus dimensiones.

7.2. Imágenes desde archivos locales y URLs.

La forma en que especificas la ruta en el atributo src depende de dónde se encuentre la imagen:

7.2.1. Archivos Locales (Rutas Relativas)

Si la imagen está en la misma carpeta que tu archivo HTML, o en una subcarpeta dentro de tu proyecto, utilizas una ruta relativa. Estas rutas son relativas a la ubicación del archivo HTML actual.

  • Misma carpeta: <img src="foto.jpg">
  • Subcarpeta: <img src="carpeta_imagenes/foto.jpg">
  • Carpeta superior: <img src="../foto.jpg"> (.. sube un nivel en la jerarquía de carpetas)
  • Carpeta superior y luego a otra: <img src="../otra_carpeta/foto.jpg">

Ejemplo: Si tu HTML está en Pages/clase.html y tu imagen en Assets/img/imagen.jpg, la ruta sería <img src="../Assets/img/imagen.jpg">.

7.2.2. Imágenes desde URLs (Rutas Absolutas)

Puedes enlazar directamente a una imagen que está alojada en otro servidor web utilizando su URL completa (Uniform Resource Locator). Esto es común para incrustar imágenes de servicios externos o si no quieres alojar las imágenes en tu propio servidor.

Ejemplo:

<img src="https://upload.wikimedia.org/wikipedia/commons/thumb/6/61/HTML5_logo_and_wordmark.svg/800px-HTML5_logo_and_wordmark.svg.png" alt="Logo de HTML5">

Consideración: Al usar URLs externas, dependes de que el servidor que aloja la imagen esté siempre disponible y no cambie la URL. Es generalmente más seguro alojar tus propias imágenes si es posible.

7.3. Buenas prácticas para el atributo alt.

El texto alternativo es crucial por varias razones:

  • Accesibilidad: Los lectores de pantalla lo utilizan para describir las imágenes a usuarios con discapacidad visual. Es la forma en que estos usuarios "ven" la imagen.
  • SEO (Optimización para Motores de Búsqueda): Los motores de búsqueda no pueden "ver" imágenes. Utilizan el texto alt para entender de qué trata la imagen, lo que puede ayudar a que tu imagen y tu página aparezcan en los resultados de búsqueda de imágenes.
  • Fallo de carga: Si la imagen no se carga (por un error en la ruta, conexión lenta, etc.), el texto alt se mostrará en su lugar, proporcionando contexto al usuario.

Consejos para un buen texto alt:

  • Sé descriptivo: Describe el contenido de la imagen de forma concisa y precisa.
  • Sé conciso: Evita descripciones excesivamente largas.
  • Contextualiza: El texto alt debe ser relevante para el contenido de la página donde se inserta la imagen.
  • Evita "Imagen de..." o "Foto de...": Los lectores de pantalla ya anuncian que es una imagen, por lo que estas frases son redundantes.
  • Imágenes decorativas: Si una imagen es puramente decorativa y no añade información al contenido (ej. un borde, un icono sin significado funcional), el atributo alt debe estar vacío (alt=""). Esto le dice al lector de pantalla que ignore la imagen. ¡Úsalo con precaución!

Ejemplos:

  • Mal: <img src="perro.jpg" alt="imagen">
  • Regular: <img src="perro.jpg" alt="Un perro">
  • Bueno: <img src="perro.jpg" alt="Perro golden retriever jugando en un parque.">
  • Mejor (si es un logo): <img src="logo.png" alt="Logotipo de [Nombre de la Empresa]">

7.4. Imágenes como enlaces.

Las imágenes no solo pueden ser elementos visuales estáticos; también pueden funcionar como enlaces, permitiendo a los usuarios hacer clic en ellas para navegar a otra página o recurso.

Para hacer que una imagen sea un enlace, simplemente envuelve la etiqueta <img> con la etiqueta de ancla <a>.

Ejemplo:

<a href="https://www.google.com" target="_blank">
    <img src="imagenes/google_logo.png" alt="Ir a Google" width="100" height="34">
</a>

En este ejemplo, al hacer clic en el logo de Google, el usuario será redirigido a la página principal de Google. El atributo target="_blank" hace que el enlace se abra en una nueva pestaña del navegador. Recuerda los atributos de seguridad que tienes que utilizar para que la página de destino no tenga acceso a tu página: rel="noopener noreferrer".

Consejo: Cuando usas una imagen como enlace, el texto alternativo del <img> se convierte en el texto del enlace para los lectores de pantalla. Asegúrate de que el alt describa el destino del enlace, no solo la imagen en sí. Por ejemplo, "Logo de Google" está bien, pero "Ir al sitio web de Google" o "Buscar en Google" podría ser aún más explícito si el logo es el único contenido clicable.
