CURSO DE INICIACIÓN A HTML

Clase 4: Elementos de Texto II: Formato y Énfasis.

En esta cuarta clase de nuestro curso de HTML, profundizaremos en cómo dar formato y énfasis al texto dentro de nuestras páginas web. Más allá de los encabezados y párrafos básicos, HTML nos ofrece una variedad de etiquetas semánticas y de presentación que nos permiten destacar información, indicar su importancia o su naturaleza especial, y mejorar la legibilidad y la accesibilidad de nuestro contenido. Comprender y aplicar estas etiquetas correctamente es fundamental para crear documentos web bien estructurados y significativos.

4.1. Énfasis de Texto: <strong> y <em>

HTML proporciona dos etiquetas principales para enfatizar el texto, cada una con un significado semántico distinto:

4.1.1. La Etiqueta <strong> (Importancia)

La etiqueta <strong> se utiliza para indicar que un fragmento de texto tiene gran importancia, seriedad o urgencia. Los navegadores suelen renderizar el texto dentro de <strong> en negrita, pero su propósito principal es comunicar el significado semántico a los agentes de usuario (como los lectores de pantalla) y a los motores de búsqueda.

Uso semántico: No uses <strong> simplemente para poner texto en negrita (para eso se usaría CSS). Úsala cuando el contenido dentro de ella sea críticamente importante o urgente.

Ejemplo:

<p>Este es un <strong>aviso importante</strong>: la fecha límite es mañana.</p>

En este ejemplo, "aviso importante" no solo se verá en negrita, sino que se le dará un peso semántico que un lector de pantalla, por ejemplo, podría anunciar con una entonación diferente.

4.1.2. La Etiqueta <em> (Énfasis Tonal)

La etiqueta <em> se utiliza para indicar un énfasis tonal o de voz, como si estuvieras leyendo el texto en voz alta y quisieras destacar ciertas palabras. Los navegadores suelen renderizar el texto dentro de <em> en cursiva.

Uso semántico: Usa <em> cuando quieras cambiar el significado de una oración al enfatizar una palabra específica, similar a cómo lo harías al hablar.

Ejemplo:

<p>Me <em>encanta</em> el HTML.</p>
<p>Me encanta el <em>HTML</em>.</p>

En el primer caso, el énfasis está en la acción de "encantar". En el segundo, el énfasis está en el "HTML" como el objeto del afecto. Ambos se verán en cursiva, pero semánticamente son diferentes.

Diferencia clave: <strong> indica importancia (el contenido es significativo por sí mismo), mientras que <em> indica énfasis (cómo se diría el contenido para alterar su significado o matiz).

4.2. Texto Más Pequeño: <small>

La etiqueta <small> se utiliza para representar comentarios al margen y letra pequeña, incluyendo derechos de autor, cláusulas legales y atribuciones. Semánticamente, indica que el contenido tiene una importancia secundaria.

Los navegadores suelen mostrar el texto dentro de <small> con un tamaño de fuente ligeramente más pequeño que el texto circundante.

Ejemplo:

<footer>
    <p><small>&copy; 2025 Mi Sitio Web. Todos los derechos reservados.</small></p>
</footer>

Es ideal para la información de derechos de autor, notas a pie de página o descargos de responsabilidad que no son parte del contenido principal pero son relevantes.

4.3. Subíndice y Superíndice: <sub>, <sup>

Estas dos etiquetas son esenciales para la notación científica, matemática y para ciertos formatos de texto específicos.

4.3.1. La Etiqueta <sub> (Subíndice)

La etiqueta <sub> se utiliza para mostrar texto como un subíndice, es decir, ligeramente por debajo de la línea normal de texto y con un tamaño de fuente reducido. Es comúnmente utilizada en fórmulas químicas.

Ejemplo:

<p>La fórmula del agua es H<sub>2</sub>O.</p>

Esto se renderizará como "H₂O".

4.3.2. La Etiqueta <sup> (Superíndice)

La etiqueta <sup> se utiliza para mostrar texto como un superíndice, es decir, ligeramente por encima de la línea normal de texto y con un tamaño de fuente reducido. Se usa frecuentemente en notación matemática (exponentes), para ordinales o para referencias de notas a pie de página.

Ejemplo:

<p>Einstein es famoso por su ecuación E=mc<sup>2</sup>.</p>
<p>El 1<sup>er</sup> premio fue para John Doe.</p>
<p>Esta es una nota al pie<sup>[1]</sup>.</p>

Esto se renderizará como "E=mc²", "1ᵉʳ" y "nota al pie¹".

4.4. Texto Marcado o Resaltado: <mark>

La etiqueta <mark> se utiliza para resaltar o "marcar" texto, indicando que es relevante en un contexto particular o para fines de referencia. Es similar a usar un marcador fluorescente en un texto impreso.

Los navegadores suelen mostrar el texto dentro de <mark> con un color de fondo resaltado (a menudo amarillo).

Uso semántico: La etiqueta <mark> es adecuada para cuando el texto está siendo citado o referenciado en el contexto actual, y se desea llamar la atención sobre él.

Ejemplo:

<p>Busca la palabra <mark>importante</mark> en el párrafo.</p>

Esto es útil en resultados de búsqueda para resaltar las palabras clave encontradas, o para destacar información crucial en un documento.

4.5. Citas en Bloque: <blockquote>

La etiqueta <blockquote> se utiliza para indicar que el texto contenido dentro de ella es una cita de otra fuente, y que esta cita debe ser presentada como un bloque de texto separado, a menudo con sangría o un formato visual distinto para distinguirla del contenido principal.

Atributo cite: Opcionalmente, puedes usar el atributo cite para proporcionar la URL de la fuente de la cita. Este atributo no es visible para el usuario, pero puede ser útil para los motores de búsqueda y herramientas de accesibilidad.

Ejemplo:

<blockquote cite="https://es.wikipedia.org/wiki/Tim_Berners-Lee">
    <p>La Web es más una creación social que una creación técnica. La diseñé para un efecto social: ayudar a la gente a colaborar, y no como una herramienta tecnológica para acciones aisladas.</p>
    <footer>— Tim Berners-Lee</footer>
</blockquote>

Los navegadores suelen aplicar sangría al contenido dentro de <blockquote> por defecto. Es una buena práctica incluir la atribución del autor dentro de la misma cita, a menudo dentro de un elemento <footer> o cite si es el título de la obra.

Diferencia con <q>: Mientras <blockquote> es para citas en bloque (que ocupan su propio párrafo o más), la etiqueta <q> es para citas cortas en línea que no requieren un salto de párrafo (por ejemplo, "El profesor dijo: Estudien mucho").
