Clase 6: Enlaces (Hyperlinks) - Curso HTML Completo

Hoy retomamos nuestro curso de HTML y, continuando con la emocionante aventura de construir páginas web interactivas, nos sumergiremos en uno de los pilares fundamentales de la web: los enlaces, o hipervínculos. Sin ellos, la web sería una colección de documentos aislados. Gracias a los enlaces, podemos navegar sin esfuerzo entre diferentes páginas, tanto dentro de un mismo sitio como a través de la vasta red global.

Clase 6: Enlaces (Hyperlinks): Navegación entre Páginas

Desvelamos el poder de la navegación en la web. Aprendemos cómo conectar tus páginas, llevar a tus usuarios a otros sitios, e incluso permitirles iniciar correos electrónicos o llamadas telefónicas directamente desde tu sitio web. Prepárense para tejer la telaraña de su propio contenido.

6.1. La Etiqueta <a> y el Atributo href

El corazón de todo enlace en HTML reside en la etiqueta <a>, que significa "anchor" (ancla). Esta etiqueta por sí sola no hace nada especial; necesita un atributo crucial para funcionar como un enlace: el atributo href.

href (Hypertext Reference): Este atributo es donde especificamos la URL (Uniform Resource Locator) a la que queremos que el enlace apunte. En otras palabras, es la dirección del destino al que el usuario será llevado al hacer clic en el enlace.

La estructura básica de un enlace es la siguiente:

<a href="URL_DEL_DESTINO">Texto del enlace</a>
  • URL_DEL_DESTINO: Aquí colocamos la dirección de la página o recurso al que queremos enlazar.
  • Texto del enlace: Este es el texto visible en la página web sobre el que el usuario hará clic. Es fundamental que este texto sea descriptivo y claro, indicando al usuario qué esperar al seguir el enlace.

Ejemplo:

<p>Visita la <a href="https://www.google.com">página principal de Google</a> para buscar información.</p>

En este ejemplo, "página principal de Google" es el texto que se mostrará como un enlace, y al hacer clic en él, el navegador llevará al usuario a https://www.google.com.

6.2. Enlaces Internos (Dentro del Mismo Sitio)

Los enlaces internos son aquellos que conectan diferentes páginas o secciones dentro de tu propio sitio web. Son esenciales para la navegación de tu sitio, permitiendo a los usuarios moverse fluidamente entre tus contenidos.

Cuando creamos enlaces internos, podemos usar dos tipos de rutas en el atributo href:

6.2.1. Rutas Relativas

Las rutas relativas son las más comunes y recomendadas para enlaces internos. Son "relativas" a la ubicación del archivo HTML actual. Esto significa que no necesitas especificar la URL completa del sitio web, solo la ruta desde la página actual hasta el archivo de destino.

Ventajas de las rutas relativas:

  • Portabilidad: Si mueves tu sitio web a un servidor diferente o a una carpeta distinta, los enlaces seguirán funcionando siempre y cuando la estructura de directorios interna se mantenga. No tendrás que actualizar todas las URLs.
  • Facilidad de mantenimiento: Si cambias el nombre de tu dominio o la URL base de tu sitio, no tendrás que modificar cada enlace interno.

Ejemplos de rutas relativas:

  • Enlace a un archivo en el mismo directorio: Si index.html y sobre_nosotros.html están en la misma carpeta: 
    <a href="sobre_nosotros.html">Conoce más sobre nosotros</a>
  • Enlace a un archivo en un subdirectorio: Si productos.html está dentro de una carpeta llamada seccion_productos que está en el mismo nivel que tu archivo actual: 
    <a href="seccion_productos/productos.html">Nuestros productos</a>
  • Enlace a un archivo en un directorio superior (o subir de nivel): Si tu archivo actual está en blog/articulo1.html y quieres enlazar a contacto.html que está en la carpeta raíz (un nivel superior): 
    <a href="../contacto.html">Contáctanos</a>
    El ../ indica "subir un nivel en la estructura de directorios". Puedes usar ../../ para subir dos niveles, y así sucesivamente.
  • Enlace a la página principal (índice) desde un subdirectorio: Si estás en seccion_productos/productos.html y quieres ir a index.html en la raíz: 
    <a href="../index.html">Ir a la página principal</a>
    Puedes tambien acceder directamente a index.html con href="/" y el intérprete ya sabe que vas a la página principal.

6.2.2. Rutas Absolutas (Dentro del Mismo Sitio)

Aunque menos comunes para enlaces internos a archivos dentro del mismo sitio, también puedes usar rutas absolutas. Estas rutas especifican la URL completa desde la raíz de tu dominio.

Ejemplo:

Si tu dominio es midominio.com y quieres enlazar a la página acerca.html que está en la raíz:

<a href="/acerca.html">Acerca de nosotros</a>

Observa que la ruta comienza con /, lo que indica la raíz del dominio. Si tu sitio está en un subdirectorio como midominio.com/mi_sitio/, entonces la ruta absoluta sería /mi_sitio/acerca.html.

Consideraciones:

  • Para sitios pequeños o en desarrollo local, las rutas relativas son generalmente más convenientes.
  • Las rutas absolutas pueden ser útiles en ciertos sistemas de gestión de contenido (CMS) o si necesitas referenciar recursos de forma consistente desde cualquier parte de tu sitio, independientemente de la ubicación del archivo actual.

6.3. Enlaces Externos (A Otros Sitios Web)

Los enlaces externos, como su nombre indica, dirigen a los usuarios a páginas o recursos ubicados en otros dominios web. Son cruciales para citar fuentes, recomendar recursos, o simplemente conectar tu sitio con el resto de la web.

Para crear un enlace externo, el atributo href debe contener la URL completa y absoluta del destino, incluyendo el protocolo (http:// o https://).

Ejemplo:

<p>Puedes encontrar más información sobre desarrollo web en <a href="https://developer.mozilla.org/es/docs/Web">MDN Web Docs</a>.</p>

Importante: Siempre incluye https:// (o http:// si el sitio no tiene SSL, aunque esto es cada vez menos común y no recomendado) al inicio de la URL para enlaces externos. De lo contrario, el navegador podría intentar interpretar la URL como una ruta relativa dentro de tu propio sitio, lo que resultaría en un enlace roto.

6.4. Abrir Enlaces en una Nueva Pestaña: target="_blank"

Por defecto, cuando un usuario hace clic en un enlace, la página actual del navegador se reemplaza por la página de destino. Sin embargo, a menudo querrás que los enlaces externos (o incluso algunos internos) se abran en una nueva pestaña o ventana del navegador, manteniendo tu sitio web original abierto en la pestaña anterior. Esto mejora la experiencia del usuario, ya que no "pierden" tu sitio al seguir un enlace.

Para lograr esto, utilizamos el atributo target con el valor _blank.

Sintaxis:

<a href="URL_DEL_DESTINO" target="_blank">Texto del enlace</a>

Ejemplo:

<p>Para ver un ejemplo de un sitio responsivo, visita <a href="https://getbootstrap.com/" target="_blank">Bootstrap (se abrirá en una nueva pestaña)</a>.</p>

Consideraciones de seguridad y rendimiento al usar target="_blank":

Aunque target="_blank" es muy útil, tiene implicaciones de seguridad y rendimiento que debes conocer:

  1. rel="noopener noreferrer": Cuando usas target="_blank", la página que se abre en la nueva pestaña puede tener acceso limitado al objeto window.opener de la página original. Esto puede ser explotado por sitios maliciosos para realizar ataques de phishing (engañar al usuario para que piense que está en tu sitio) o para consumir recursos de tu página. Para mitigar este riesgo, es una práctica recomendada y casi obligatoria añadir los atributos rel="noopener noreferrer" (o al menos rel="noopener") al mismo tiempo que target="_blank".
  • noopener: Evita que la nueva página pueda acceder a la propiedad window.opener.
  • noreferrer: Además de noopener, evita que el navegador envíe el Referer HTTP header, lo que significa que la página de destino no sabrá desde qué página llegó el usuario. Esto puede ser útil para la privacidad del usuario, aunque algunos servicios de análisis podrían no registrar la fuente del tráfico.

Sintaxis recomendada para enlaces externos que abren en una nueva pestaña:

<a href="URL_DEL_DESTINO" target="_blank" rel="noopener noreferrer">Texto del enlace</a>

Ejemplo con seguridad:

<p>Consulta la documentación oficial de <a href="https://developer.mozilla.org/es/docs/Web/HTML" target="_blank" rel="noopener noreferrer">HTML en MDN (nueva pestaña segura)</a>.</p>

Es una buena práctica implementar rel="noopener noreferrer" cada vez que uses target="_blank" para proteger a tus usuarios y tu propio sitio.

6.5. Enlaces de Correo Electrónico y Teléfono

Además de enlazar a otras páginas web, la etiqueta <a> también puede utilizarse para iniciar acciones específicas en el sistema del usuario, como abrir su cliente de correo electrónico o iniciar una llamada telefónica.

6.5.1. Enlaces de Correo Electrónico (mailto:)

Puedes crear un enlace que, al hacer clic, abra el cliente de correo electrónico predeterminado del usuario (como Outlook, Thunderbird o Gmail en el navegador) y pre-rellene la dirección del destinatario.

Para esto, el valor del atributo href comienza con mailto: seguido de la dirección de correo electrónico.

Sintaxis:

<a href="mailto:DIRECCION_DE_CORREO">Texto del enlace</a>

Ejemplo:

<p>¿Tienes preguntas? <a href="mailto:info@tudominio.com">Envíanos un correo electrónico</a>.</p>

Opciones adicionales con mailto::

Puedes añadir parámetros a la URL mailto: para pre-rellenar otros campos del correo electrónico, como el asunto, el cuerpo del mensaje o incluso destinatarios en copia (CC) o copia oculta (BCC). Los parámetros se añaden después de la dirección de correo electrónico, comenzando con ? para el primer parámetro y & para los subsiguientes.

  • subject (Asunto): 
    <a href="mailto:info@tudominio.com?subject=Consulta%20desde%20la%20web">Enviar consulta con asunto</a>
    (Nota: Los espacios se representan con %20 o un + en las URLs, aunque el navegador suele manejarlos bien, es una buena práctica codificarlos).
  • body (Cuerpo del mensaje): 
    <a href="mailto:info@tudominio.com?body=Me%20gustaria%20saber%20mas%20sobre...">Enviar con cuerpo predefinido</a>
  • cc (Copia de carbón): 
    <a href="mailto:info@tudominio.com?cc=ventas@tudominio.com">Enviar a info y ventas</a>
  • bcc (Copia de carbón oculta): 
    <a href="mailto:info@tudominio.com?bcc=privado@tudominio.com">Enviar con copia oculta</a>
  • Combinando varios parámetros: 
    <a href="mailto:info@tudominio.com?subject=Contacto%20web&body=Hola,%20me%20pon%20en%20contacto%20desde%20su%20sitio%20web.">Contactar con asunto y cuerpo</a>

6.5.2. Enlaces de Teléfono (tel:)

En dispositivos móviles, puedes crear enlaces que, al hacer clic, inicien una llamada telefónica al número especificado.

Para esto, el valor del atributo href comienza con tel: seguido del número de teléfono.

Sintaxis:

<a href="tel:NUMERO_DE_TELEFONO">Texto del enlace</a>

Ejemplo:

<p>Llámanos ahora: <a href="tel:+34912345678">Teléfono: +34 912 34 56 78</a></p>

Recomendaciones para números de teléfono:

  • Incluye el código de país (ej., +34 para España) para asegurarte de que funcione globalmente.
  • Evita espacios, guiones o paréntesis en el número dentro del href para una mayor compatibilidad, aunque los navegadores modernos suelen ser flexibles. Sin embargo, en el texto visible del enlace, puedes formatearlo para que sea más legible para los usuarios.

6.6. Enlaces a Secciones Específicas (Anclas con ID)

Aunque esto es más avanzado, es fundamental para una buena navegación interna. Puedes enlazar a una sección específica dentro de la misma página HTML o en otra página, utilizando el atributo id.

  1. Definir un ancla: Asigna un id único a cualquier elemento HTML al que quieras enlazar. 
    <h2 id="seccion-contacto">Contacto</h2>
<p>Puedes rellenar el formulario de contacto...</p>
  2. Crear el enlace: En el href, usa # seguido del id del elemento.
    • Dentro de la misma página: 
      <p><a href="#seccion-contacto">Ir a la sección de Contacto</a></p>
      Al hacer clic, la página se desplazará suavemente hasta el encabezado "Contacto".
    • A una sección en otra página: 
      <a href="sobre_nosotros.html#mision">Ver nuestra misión en la página "Sobre Nosotros"</a>
      Esto cargará sobre_nosotros.html y luego se desplazará a la sección con id="mision".
