CURSO DE INICIACIÓN A HTML

Clase 9: Comentarios HTML y Convenciones de Codificación.

A medida que tus proyectos web crecen en complejidad, la legibilidad y el mantenimiento del código se vuelven tan importantes como su funcionalidad. En esta clase, aprenderemos a utilizar comentarios en HTML para documentar nuestro código y las convenciones de codificación que hacen que nuestro trabajo sea más profesional y fácil de entender para nosotros mismos y para otros desarrolladores.

9.1. Cómo añadir comentarios en HTML <!-- ... -->.

Los comentarios son notas que los desarrolladores añaden al código fuente de un documento. Son ignorados por el navegador y no se muestran en la página web final, pero son visibles cuando se visualiza el código fuente de la página.

En HTML, los comentarios se escriben de la siguiente manera:

<!-- Este es un comentario en HTML -->

Un comentario comienza con <!-- y termina con -->. Puedes escribir cualquier texto entre ellos.

Ejemplo de uso de comentarios:

<!-- Sección del encabezado global del sitio -->
<header>
    <h1>Mi Sitio Web</h1>
    <!-- Navegación principal -->
    <nav>
        <ul>
            <li><a href="/">Inicio</a></li>
            <li><a href="/servicios">Servicios</a></li>
            <!-- Enlace a la página de contacto (pendiente de creación) -->
            <li><a href="/contacto">Contacto</a></li>
        </ul>
    </nav>
</header>

Importante: Los comentarios HTML no pueden anidarse. Es decir, no puedes poner un comentario dentro de otro comentario. Si lo intentas, el navegador puede interpretar el código de forma inesperada.

<!-- Este es el comentario padre <!-- Este es un comentario anidado (¡incorrecto!) --> -->

Evita usar -- dentro del texto de un comentario, ya que esto podría ser interpretado como el cierre prematuro del comentario.

9.2. La importancia de los comentarios para la legibilidad del código.

Aunque los comentarios no afectan directamente el funcionamiento de tu página web, son una herramienta invaluable para los desarrolladores por varias razones:

  • Claridad y comprensión: Ayudan a explicar secciones complejas o no obvias del código. Cuando tú (o alguien más) revises el código en el futuro, los comentarios pueden refrescar la memoria sobre el propósito de una sección específica.
  • Documentación: Sirven como una forma de documentación interna, describiendo la lógica detrás de ciertas decisiones de diseño o por qué un fragmento de código está estructurado de una manera particular.
  • Depuración y desarrollo: Puedes "comentar" temporalmente bloques de código para depurar problemas o probar diferentes implementaciones sin borrarlos por completo. Esto es muy útil durante el desarrollo.
  • Colaboración: Facilita el trabajo en equipo, ya que los comentarios permiten a los miembros del equipo entender rápidamente las contribuciones de los demás y seguir un estilo de codificación común.
  • Notas para el futuro: Puedes dejar notas para ti mismo sobre tareas pendientes, mejoras futuras o advertencias sobre el código.

¿Cuándo usar comentarios?

  • Para describir el propósito de secciones grandes de tu página (ej., ``).
  • Para explicar etiquetas o atributos inusuales o complejos.
  • Para dejar notas sobre cambios futuros o problemas conocidos.
  • Para ocultar temporalmente código que no quieres que se renderice.

Consejo: No abuses de los comentarios. Un código bien escrito y semántico a menudo se explica por sí mismo. Comenta aquello que no es obvio o que requiere una explicación adicional.

9.3. Convenciones básicas de indentación y formato.

La forma en que organizas visualmente tu código se conoce como convenciones de codificación o estilo de código. Aunque HTML es indulgente con el espacio en blanco y las nuevas líneas, seguir estas convenciones es crucial para la legibilidad y la profesionalidad:

9.3.1. Indentación

La indentación es el uso de espacios o tabulaciones para empujar los elementos anidados hacia la derecha. Esto crea una jerarquía visual que refleja la estructura anidada del HTML.

  • Recomendado: Indentar con 2 o 4 espacios, o con tabulaciones. La consistencia es clave. Si usas espacios, úsalos siempre. Si usas tabulaciones, úsalas siempre. Los espacios suelen ser preferidos porque aseguran que el código se vea igual en cualquier editor, independientemente de la configuración de tabulación.
  • Cada vez que un elemento está anidado dentro de otro, se debe indentar.

Ejemplo (buena indentación):

<body>
    <header>
        <h1>Título de la Página</h1>
        <nav>
            <ul>
                <li><a href="#">Enlace 1</a></li>
                <li><a href="#">Enlace 2</a></li>
            </ul>
        </nav>
    </header>
</body>

Ejemplo (mala indentación - difícil de leer):

<body>
<header>
<h1>Título de la Página</h1>
<nav>
<ul>
<li><a href="#">Enlace 1</a></li>
<li><a href="#">Enlace 2</a></li>
</ul>
</nav>
</header>
</body>

9.3.2. Uso de Mayúsculas/Minúsculas

Aunque HTML no distingue entre mayúsculas y minúsculas para las etiquetas y atributos (<DIV> es lo mismo que <div>), la convención y la recomendación es usar siempre minúsculas.

Ejemplo (correcto): <div> <p> <a href="#">

Ejemplo (incorrecto/no recomendado): <DIV> <P> <A HREF="#">

9.3.3. Comillas en Atributos

Siempre encierra los valores de los atributos entre comillas (dobles o simples). Las comillas dobles (") son las más comunes.

Ejemplo (correcto): <img src="imagen.jpg" alt="Descripción">

Ejemplo (incorrecto/no recomendado): <img src=imagen.jpg alt=Descripción> (Aunque a veces funcione, no es una buena práctica y puede causar problemas.)

9.3.4. Espacios

Usa un espacio después de la etiqueta en el atributo (ej., <img src="..." alt="...">, no <imgsrc="..."alt="...">).

Seguir estas convenciones, junto con el uso estratégico de comentarios, transformará tu código de HTML funcional a un HTML legible, mantenible y profesional.
