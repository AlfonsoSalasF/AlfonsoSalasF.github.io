Guía: Configuración y Sincronización de Proyecto Local con GitHub

Esta guía resume el proceso paso a paso para configurar tu proyecto local (organizado con las carpetas: CajónDesastre, CSS, Ejercicios, Images, Pages y index.html en la raíz) con un repositorio existente en GitHub, asegurando una sincronización efectiva.

Consideración importante: Este proceso asume que estás de acuerdo en que el contenido de tu repositorio en GitHub (AlfonsoSalasF.github.io) será sobrescrito por el contenido de tu carpeta local.

Preparación Inicial (en tu máquina local)

  • Asegúrate de que la estructura de tu proyecto local sea exactamente como la deseas en la raíz de tu carpeta principal: 
    
tu_proyecto_local/
├── CajónDesastre/
├── CSS/
│   └── tu_estilo.css  (ejemplo)
├── Ejercicios/
├── Images/
│   └── LogoHTML5.png  (ejemplo)
├── Pages/
│   └── 01IntroduccionAlDesarrolloWebYHtml.html (ejemplo)
└── index.html
  • Abre tu carpeta tu_proyecto_local en Visual Studio Code.

Paso 1: Inicializar Git en tu Proyecto Local

Abre la terminal integrada en VS Code (Terminal > New Terminal o Ctrl+Ñ).

  • **Verifica si Git ya está inicializado:** 
    git status
    Si la respuesta es "not a git repository", continúa con el siguiente punto. Si ya es un repositorio, asegúrate de que no haya cambios pendientes o que sean irrelevantes para este proceso.
  • **Inicializa el repositorio Git (si es necesario):** 
    git init
    Esto crea la carpeta oculta .git para el seguimiento de versiones.

Paso 2: Realizar el Primer Commit Local

  • **Crea o actualiza tu archivo .gitignore (Opcional pero Recomendado):**

    Crea un archivo llamado .gitignore (sin extensión, solo .gitignore) en la raíz de tu proyecto local.

    Puedes crearlo así:

    • **Desde VS Code:** Clic derecho en el explorador de archivos (en la raíz del proyecto) > "New File..." > Escribe .gitignore.
    • **Desde la Terminal:** Asegúrate de estar en la raíz de tu proyecto y ejecuta: 
      touch .gitignore

    Este archivo le dice a Git qué archivos y carpetas debe ignorar (no rastrear ni incluir en los commits), como archivos temporales o de configuración local.

  • **Añadir todos tus archivos al "Staging Area":**

    El "staging area" es un área intermedia donde preparas los cambios que quieres incluir en tu próximo commit. No guarda los cambios directamente, sino que los selecciona.

    git add .
    Esto prepara todos los archivos y carpetas de tu proyecto local para el commit. En VS Code, verás los archivos listados en el panel de "Control de Código Fuente", listos para ser confirmados.
  • **Realizar el commit local:**

    Un commit es una "instantánea" de tu proyecto en un momento dado, guardada en tu historial de Git local.

    git commit -m "Initial project setup and structure"
    El mensaje entre comillas es descriptivo de los cambios.

Paso 3: Conectar tu Repositorio Local con GitHub

  • **Quitar la conexión remota existente (si la hay):**

    Esto previene conflictos si tu repositorio local estaba conectado a una URL antigua o incorrecta.

    git remote remove origin
    (Si obtienes un error como "No such remote", ignóralo, significa que no había ninguna conexión remota configurada).
  • **Añadir la conexión a tu repositorio existente en GitHub:**

    Necesitas la URL HTTPS de tu repositorio AlfonsoSalasF.github.io. Haz clic en el botón verde "Code" en tu repositorio de GitHub para copiarla.

    git remote add origin https://github.com/AlfonsoSalasF/AlfonsoSalasF.github.io.git
    ¡Importante!: Siempre añade .git al final de la URL al usar git remote add.
  • **Establecer la rama principal y forzar la subida a GitHub:**

    Estos comandos renombran tu rama principal local a main (el estándar actual de GitHub) y luego fuerzan la subida de todo tu contenido local al repositorio remoto, sobrescribiendo lo que haya allí.

    git branch -M main
    git push -u origin main --force
    ¡ADVERTENCIA CRÍTICA del --force! Este comando reescribirá completamente el historial y el contenido de tu rama main en GitHub con lo que tienes localmente. Cualquier archivo que estuviera en GitHub y no en tu copia local se perderá. Úsalo solo si estás seguro de que el contenido local es el que deseas que sea la "verdad".

    Cuando ejecutes el git push, es probable que te pida autenticación:

    • **"Connect to GitHub Browser" (Recomendado):** Es la opción más fácil y segura. Se abrirá tu navegador para que inicies sesión y autorices la conexión.
    • **"Token":** Requiere que generes un "Personal Access Token" (PAT) en la configuración de tu cuenta de GitHub y lo pegues en la terminal. Más pasos, pero útil para automatización.

Paso 4: Flujo de Trabajo para Sincronización Continua

Una vez configurado, este es el proceso diario para mantener tu proyecto local y GitHub sincronizados.

Cuando haces cambios en VS Code (localmente):

  1. **Guarda tus archivos.**
  2. **Prepara los cambios (Stage Changes):**
    • Desde el panel de "Control de Código Fuente" en VS Code: Haz clic en el + junto a los archivos modificados o en "Stage All Changes".
    • Desde la Terminal: 
      git add .
  3. **Confirma los cambios (Commit Changes):**
    • Desde el panel de "Control de Código Fuente": Escribe un mensaje descriptivo en el cuadro de texto y haz clic en "Commit".
    • Desde la Terminal: 
      git commit -m "Tu mensaje descriptivo aquí"
  4. **Descarga y Sube los cambios (Pull antes de Push):**

    Es buena práctica descargar cambios del remoto (pull) antes de subir los tuyos (push) para evitar conflictos, incluso si eres el único trabajando.

    • Desde VS Code: Haz clic en el botón "Sync Changes" en la barra de estado inferior, que realiza un pull y luego un push. O clic en ... en "Control de Código Fuente" y selecciona "Pull" y luego "Push".
    • Desde la Terminal: Primero 
      git pull origin main
      y luego 
      git push origin main

Cuando haces cambios directamente en GitHub.com:

  1. Edita los archivos directamente en la interfaz web de GitHub.
  2. Confirma los cambios ("Commit changes") en GitHub.
  3. **Descarga los cambios en tu VS Code (Pull Changes):**

    Cuando vuelvas a tu proyecto local, actualiza tu copia:

    • Desde VS Code: Haz clic en el botón "Sync Changes" en la barra de estado, o en ... en "Control de Código Fuente" y selecciona "Pull".
    • Desde la Terminal: 
      git pull origin main

Conceptos Clave y Consideraciones

  • **.gitignore:** Archivo en la raíz de tu proyecto (.gitignore sin extensión) que lista archivos/carpetas a ignorar por Git (no rastrear ni subir).
  • **Staging Area:** Área intermedia donde preparas los cambios para el próximo commit. git add . los mueve aquí.
  • **Commit:** "Guardar" una instantánea de los cambios preparados en el historial de tu repositorio local.
  • **Push:** Enviar los commits de tu repositorio local al repositorio remoto en GitHub.
  • **Pull:** Descargar los commits del repositorio remoto (GitHub) a tu repositorio local.
  • **Eliminar archivos:** Borrar un archivo localmente no lo elimina de GitHub hasta que lo git add, git commit y git push. Git te da control explícito sobre las eliminaciones.
