Clase 15: Botones y Campos de Texto Multilínea.

En esta clase, nos centraremos en dos elementos de formulario esenciales: las áreas de texto multilínea para entradas extensas y los diferentes tipos de botones que permiten a los usuarios interactuar con los formularios, enviándolos, restableciéndolos o activando scripts.

15.1. <textarea> para entradas de texto multilínea.

A diferencia de <input type="text"> que es para una sola línea, la etiqueta <textarea> se utiliza para crear un campo de entrada donde el usuario puede escribir varias líneas de texto. Es ideal para comentarios, mensajes largos, descripciones, etc.

Atributos comunes:

  • name: El nombre del control que se enviará al servidor.
  • id: Identificador único para asociarlo con una <label>.
  • rows: (Obligatorio para la validación HTML5, pero se puede omitir y usar CSS) Especifica el número visible de líneas de texto.
  • cols: (Obligatorio para la validación HTML5, pero se puede omitir y usar CSS) Especifica el ancho visible en número de caracteres promedio.
  • placeholder: Un texto de pista que se muestra cuando el área de texto está vacía.
  • readonly, disabled, required: Funcionan de manera similar a los campos <input>.

Ejemplo:

<form>
    <label for="comentarios">Tus comentarios:</label>
    <textarea id="comentarios" name="comentarios" rows="5" cols="40" placeholder="Escribe aquí tus sugerencias..."></textarea>
    <br><br>
    <label for="descripcion_producto">Descripción del producto:</label>
    <textarea id="descripcion_producto" name="descripcion_producto" rows="10" placeholder="Describe el producto en detalle..." required></textarea>
    <button type="submit">Enviar</button>
</form>


Tamaño del textarea: Aunque rows y cols controlan el tamaño inicial, el tamaño final a menudo se maneja mejor con CSS (propiedades width y height), y se puede permitir que el usuario redimensione el campo con la propiedad CSS resize.

15.2. Botones: <button type="submit">, <button type="reset">, <button type="button">.

La etiqueta <button> es la forma más moderna y flexible de crear botones en formularios HTML. Permite incluir contenido HTML (imágenes, etiquetas, etc.) dentro del botón, a diferencia de <input type="submit">.

15.2.1. <button type="submit">

Es el tipo de botón por defecto si no se especifica. Envía los datos del formulario al servidor cuando se hace clic en él.

Ejemplo:

<form action="#" method="POST">
    <label for="nombre_btn_submit">Nombre:</label>
    <input type="text" id="nombre_btn_submit" name="nombre_btn_submit">
    <button type="submit">Enviar Formulario</button>
</form>

15.2.2. <button type="reset">

Restablece todos los campos del formulario a sus valores iniciales (los definidos por los atributos value o checked).

Ejemplo:

<form>
    <label for="campo_reset">Escribe algo:</label>
    <input type="text" id="campo_reset" name="campo_reset" value="Valor inicial">
    <button type="reset">Restablecer Formulario</button>
</form>

15.2.3. <button type="button">

No tiene comportamiento predeterminado asociado a los formularios. Se utiliza típicamente con JavaScript para activar funciones personalizadas, como mostrar/ocultar elementos, enviar datos con AJAX, etc.

Ejemplo:

<button type="button" onclick="alert('Botón clicado!')">Haz Clic</button>

Uso de alert(): En un entorno real, se preferiría usar un modal o un mensaje dentro de la página en lugar de alert() para una mejor experiencia de usuario.

15.3. El <input type="submit"> y <input type="reset">.

Estos son los equivalentes de los tipos de botón submit y reset, pero implementados como elementos <input>. La principal diferencia es que no pueden contener contenido HTML complejo, y su texto se define con el atributo value.

15.3.1. <input type="submit">

Crea un botón que envía el formulario. El texto del botón se define en su atributo value.

Ejemplo:

<form action="#" method="POST">
    <label for="usuario_input_submit">Usuario:</label>
    <input type="text" id="usuario_input_submit" name="usuario_input_submit">
    <input type="submit" value="Enviar Datos">
</form>

15.3.2. <input type="reset">

Crea un botón que restablece el formulario. El texto del botón se define en su atributo value.

Ejemplo:

<form>
    <label for="email_input_reset">Email:</label>
    <input type="email" id="email_input_reset" name="email_input_reset" value="ejemplo@dominio.com">
    <input type="reset" value="Limpiar Campos">
</form>

<button> vs. <input>: Generalmente se prefiere <button> sobre <input type="submit">/<input type="reset"> por su mayor flexibilidad en el contenido y el estilo, aunque ambos siguen siendo válidos.
