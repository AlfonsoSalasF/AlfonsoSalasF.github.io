CURSO DE INICIACIÓN A HTML

Clase 12: Elementos de Entrada de Texto.

Una vez comprendidos los conceptos básicos de los formularios en HTML, es momento de adentrarnos en los elementos de entrada más comunes: los campos de texto. El elemento <input>, con su atributo type, es increíblemente versátil y nos permite recopilar una amplia variedad de información textual del usuario. En esta clase, exploraremos los tipos de entrada de texto más utilizados y los atributos clave que nos permiten controlarlos y validarlos.

12.1. Tipos de entrada de texto: text, password, email, number, tel, url, search.

Todos estos tipos se crean utilizando la etiqueta <input> y especificando el valor de su atributo type.

12.1.1. <input type="text">

Es el tipo de entrada por defecto y el más genérico. Se utiliza para recopilar texto simple de una sola línea.

Uso común: Nombres, apellidos, direcciones (línea 1), títulos cortos.

Ejemplo:

<form>
    <label for="nombre">Nombre Completo:</label>
    <input type="text" id="nombre" name="nombre">
</form>

12.1.2. <input type="password">

Diseñado para campos de contraseña. El texto ingresado se oculta (generalmente con asteriscos o puntos) para proteger la privacidad del usuario.

Uso común: Contraseñas de inicio de sesión, campos de confirmación de contraseña.

Ejemplo:

<form>
    <label for="contrasena">Contraseña:</label>
    <input type="password" id="contrasena" name="contrasena">
</form>

12.1.3. <input type="email">

Diseñado para direcciones de correo electrónico. Los navegadores y dispositivos móviles a menudo ajustan el teclado para facilitar la entrada de correos electrónicos (ej., incluyendo el símbolo "@"). Proporciona validación básica del formato del email en el navegador.

Uso común: Campos de email en formularios de registro, contacto o boletines.

Ejemplo:

<form>
    <label for="correo">Correo Electrónico:</label>
    <input type="email" id="correo" name="correo">
</form>

12.1.4. <input type="number">

Para la entrada de números. Los navegadores suelen mostrar controles de "subir/bajar" (spinners) junto al campo y pueden restringir la entrada a solo dígitos.

Uso común: Cantidades, edades, códigos postales (si solo son números).

Ejemplo:

<form>
    <label for="edad">Edad:</label>
    <input type="number" id="edad" name="edad" min="0" max="120" step="1">
</form>

Atributos adicionales para type="number":

  • min: Valor mínimo permitido.
  • max: Valor máximo permitido.
  • step: El incremento o decremento de los controles (por defecto es 1).

12.1.5. <input type="tel">

Para números de teléfono. Aunque no valida el formato del teléfono, los navegadores en dispositivos móviles suelen invocar un teclado numérico optimizado para la entrada de números de teléfono.

Uso común: Campos de número de teléfono.

Ejemplo:

<form>
    <label for="telefono">Teléfono:</label>
    <input type="tel" id="telefono" name="telefono">
</form>

Validación de formato: Para una validación de formato específica (ej., NIF/NIE, formato de teléfono), a menudo se utiliza el atributo pattern (que veremos en clases posteriores) o JavaScript.

12.1.6. <input type="url">

Para direcciones URL. Los navegadores pueden proporcionar validación básica del formato de la URL.

Uso común: Campos de sitio web, enlaces a perfiles sociales.

Ejemplo:

<form>
    <label for="website">Tu Sitio Web:</label>
    <input type="url" id="website" name="website">
</form>

12.1.7. <input type="search">

Para campos de búsqueda. Los navegadores pueden añadir una "X" para borrar el contenido en algunos sistemas operativos, y los estilos predeterminados pueden variar.

Uso común: Barras de búsqueda en sitios web.

Ejemplo:

<form>
    <label for="buscar">Buscar en el sitio:</label>
    <input type="search" id="buscar" name="buscar">
</form>

12.2. Atributos comunes de los elementos de entrada: name, value, placeholder, readonly, disabled, required.

12.2.1. Atributo name

El atributo name es crucial. Define el nombre del control del formulario. Es el nombre que se utilizará para identificar el dato cuando el formulario se envía al servidor.

Sin un atributo name, el valor de un campo de entrada no se enviará.

Ejemplo: Si el usuario escribe "Juan" en un campo con name="nombre", el servidor recibirá "nombre=Juan".

<input type="text" name="nombre_de_usuario">

12.2.2. Atributo value

Define el valor inicial del campo de entrada. Para campos de texto, es el texto que aparece por defecto. Para otros tipos de entrada, puede ser el valor que se envía al servidor cuando el campo está seleccionado o se activa.

Ejemplo:

<input type="text" name="ciudad" value="Madrid">

12.2.3. Atributo placeholder

Proporciona una pista corta (generalmente un texto de ejemplo o una breve descripción del formato esperado) que se muestra dentro del campo de entrada cuando este está vacío. El texto desaparece automáticamente cuando el usuario comienza a escribir.

Ejemplo:

<input type="email" name="correo" placeholder="ejemplo@dominio.com">

No sustituye a <label>: El placeholder no debe usarse como sustituto de la etiqueta <label>, ya que los lectores de pantalla no lo anuncian de la misma manera y desaparece al escribir.

12.2.4. Atributo readonly

Cuando está presente, el campo de entrada no puede ser modificado por el usuario. Sin embargo, su valor se sigue enviando con el formulario.

Ejemplo:

<input type="text" name="id_usuario" value="12345" readonly>

12.2.5. Atributo disabled

Cuando está presente, el campo de entrada está inhabilitado, es decir, no es interactivo. No puede ser modificado por el usuario y, crucialmente, su valor no se envía con el formulario.

Ejemplo:

<input type="text" name="campo_inactivo" value="No editable" disabled>

12.2.6. Atributo required

Cuando está presente, indica que el campo debe ser rellenado antes de que el formulario pueda ser enviado. Los navegadores muestran un mensaje de error si el usuario intenta enviar el formulario sin rellenar un campo `required`.

Ejemplo:

<input type="text" name="apellido" required>

Validación del lado del cliente: El atributo required proporciona validación del lado del cliente (en el navegador). Siempre se debe realizar también validación del lado del servidor para garantizar la seguridad y la integridad de los datos.
