Clase 16: Estructura de Formularios: <fieldset> y <legend>.

A medida que los formularios se vuelven más complejos, es crucial organizarlos de manera lógica para mejorar tanto la usabilidad para los usuarios como la accesibilidad para las tecnologías de asistencia. En esta clase, aprenderemos sobre las etiquetas <fieldset> y <legend>, que nos permiten agrupar elementos relacionados en un formulario.

16.1. Agrupar elementos de formulario con <fieldset>.

La etiqueta <fieldset> se utiliza para agrupar semánticamente varios controles de formulario y etiquetas relacionadas dentro de un formulario. Al agrupar elementos visual y lógicamente, facilita la comprensión y navegación de formularios largos o complejos.

Visualmente, <fieldset> suele dibujar un borde alrededor de los elementos agrupados, pero su valor principal es semántico y para la accesibilidad.

Ejemplo:

<form action="#" method="POST">
    <fieldset>
        <!-- Elementos relacionados con información personal -->
        <label for="nombre_personal">Nombre:</label>
        <input type="text" id="nombre_personal" name="nombre_personal">

        <label for="apellido_personal">Apellido:</label>
        <input type="text" id="apellido_personal" name="apellido_personal">
    </fieldset>

    <fieldset>
        <!-- Elementos relacionados con información de contacto -->
        <label for="email_contacto">Email:</label>
        <input type="email" id="email_contacto" name="email_contacto">

        <label for="telefono_contacto">Teléfono:</label>
        <input type="tel" id="telefono_contacto" name="telefono_contacto">
    </fieldset>
    <button type="submit">Enviar Formulario</button>
</form>

16.2. Título del grupo con <legend>.

La etiqueta <legend> se utiliza para proporcionar un título o subtítulo para un grupo de elementos dentro de un <fieldset>. Debe ser el primer elemento dentro de <fieldset>.

Este título es esencial para dar contexto al grupo de campos, especialmente para usuarios de lectores de pantalla, que pueden navegar por grupos de campos en lugar de campo por campo.

Ejemplo:

<form action="#" method="POST">
    <fieldset>
        <legend>Datos Personales</legend>
        <label for="nombre_legend">Nombre:</label>
        <input type="text" id="nombre_legend" name="nombre_legend">

        <label for="fecha_nac_legend">Fecha de Nacimiento:</label>
        <input type="date" id="fecha_nac_legend" name="fecha_nac_legend">
    </fieldset>

    <fieldset>
        <legend>Información de Cuenta</legend>
        <label for="usuario_legend">Nombre de Usuario:</label>
        <input type="text" id="usuario_legend" name="usuario_legend">

        <label for="pass_legend">Contraseña:</label>
        <input type="password" id="pass_legend" name="pass_legend">
    </fieldset>
    <button type="submit">Registrar</button>
</form>

16.3. Mejorar la semántica y la accesibilidad de los formularios.

El uso de <fieldset> y <legend> no es meramente estético; es fundamental para la semántica y la accesibilidad de los formularios:

  • Semántica: Indica claramente qué campos están relacionados entre sí, haciendo el HTML más estructurado y comprensible tanto para los desarrolladores como para los navegadores.
  • Accesibilidad: Para los usuarios de lectores de pantalla, cuando navegan por un formulario, el lector de pantalla anunciará el texto de la <legend> al entrar en un <fieldset>. Esto proporciona un contexto crucial para el grupo de campos, ayudando a los usuarios con discapacidad visual a entender la estructura del formulario y el propósito de los campos agrupados.
  • Usabilidad: Incluso para usuarios sin discapacidades visuales, la agrupación visual de campos relacionados con un título claro mejora la legibilidad y la facilidad de uso de formularios largos.

Diseño CSS: Aunque <fieldset> y <legend> tienen estilos predeterminados (un borde y el título superpuesto), estos se pueden personalizar completamente con CSS para que se adapten al diseño de tu sitio web, manteniendo sus beneficios semánticos y de accesibilidad.
