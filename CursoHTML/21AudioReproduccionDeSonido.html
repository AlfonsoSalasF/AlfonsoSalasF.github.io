CURSO DE INICIACIÓN A HTML

Clase 21: Audio: Reproducción de Sonido

HTML5 introdujo la etiqueta <audio>, que permite incrustar contenido de audio directamente en las páginas web sin necesidad de plugins externos como Flash. Esto ha simplificado enormemente la inclusión de sonido y música, mejorando la experiencia del usuario y la compatibilidad entre dispositivos.

21.1. La etiqueta <audio>.

La etiqueta <audio> se utiliza para incrustar contenido de sonido, como música o grabaciones de voz, en un documento HTML. Es un elemento de nivel de bloque que por sí solo no muestra ningún control de reproducción a menos que se especifique el atributo controls.

Ejemplo Básico:

<audio src="audio/ejemplo.mp3" controls>
    Tu navegador no soporta el elemento de audio.
</audio>

El texto dentro de la etiqueta <audio> (en este caso, "Tu navegador no soporta el elemento de audio.") se mostrará solo si el navegador del usuario no es compatible con la etiqueta <audio>.

21.2. Atributos de <audio>.

La etiqueta <audio> soporta varios atributos que permiten controlar el comportamiento de la reproducción y la apariencia del reproductor.

Atributos Comunes de la Etiqueta <audio>
Atributo Descripción Ejemplo
src Especifica la URL del archivo de audio que se va a incrustar. Es el método más sencillo para definir la fuente. <audio src="cancion.mp3">
controls Muestra los controles de reproducción estándar del navegador (como el botón de reproducir/pausar, la barra de volumen y el control de progreso). Si no se incluye, el audio se reproducirá sin controles visibles a menos que se controle con JavaScript. <audio src="sonido.wav" controls>
autoplay Indica que el audio debe comenzar a reproducirse automáticamente tan pronto como sea posible. Se debe usar con precaución, ya que puede ser intrusivo para los usuarios. Muchos navegadores modernos bloquean la reproducción automática con sonido. <audio src="fondo.ogg" autoplay>
loop Especifica que el audio debe reproducirse en bucle, es decir, se reiniciará automáticamente cada vez que llegue al final. <audio src="ambiente.mp3" loop>
muted Indica que el audio debe silenciarse por defecto al cargarse. El usuario puede desmutearlo manualmente a través de los controles si están presentes. <audio src="intro.mp3" muted>
preload Define cómo se debe cargar el archivo de audio al cargar la página. Puede tener los siguientes valores:
  • none: El audio no se precarga.
  • metadata: Solo se precargan los metadatos (duración, etc.).
  • auto: El navegador decide si precarga el audio completo (valor por defecto).
 <audio src="podcast.mp3" preload="none">

Ejemplo de atributos combinados:

<audio src="audio/musica_fondo.mp3" autoplay loop muted>
    Tu navegador no soporta el elemento de audio.
</audio>

21.3. Formatos de audio compatibles.

No todos los navegadores soportan los mismos formatos de audio. Los formatos más comunes y recomendados para la web son:

  • MP3 (MPEG-1 Audio Layer III): Es el formato más ampliamente soportado y ofrece una buena calidad con tamaños de archivo relativamente pequeños.
  • WAV (Waveform Audio File Format): Formato sin compresión, lo que resulta en archivos de muy alta calidad pero también de gran tamaño. No es ideal para streaming web.
  • OGG (Ogg Vorbis): Un formato de código abierto y libre de patentes que ofrece buena compresión y calidad. Es un buen complemento para MP3, ya que algunos navegadores pueden tener mejor soporte para OGG.
  • AAC (Advanced Audio Coding): Utilizado a menudo en dispositivos Apple y en la codificación de video (como en archivos MP4). Ofrece buena calidad con una compresión eficiente.

Para asegurar la máxima compatibilidad, es una buena práctica proporcionar el audio en múltiples formatos, lo cual se logra con la etiqueta <source>.

21.4. Múltiples fuentes con <source>.

Para ofrecer compatibilidad con diferentes navegadores y sus distintos soportes de formatos, puedes usar la etiqueta <source> dentro de <audio>. El navegador intentará cargar el primer formato que reconozca y pueda reproducir.

La etiqueta <source> tiene dos atributos importantes:

  • src: La URL del archivo de audio.
  • type: El tipo MIME del archivo de audio (por ejemplo, "audio/mpeg" para MP3, "audio/ogg" para OGG, "audio/wav" para WAV). Esto permite al navegador decidir si puede reproducir el archivo antes de intentar descargarlo.

Ejemplo con múltiples fuentes:

<audio controls>
    <source src="audio/musica.ogg" type="audio/ogg">
    <source src="audio/musica.mp3" type="audio/mpeg">
    <!-- Texto alternativo si el navegador no soporta <audio> o ninguno de los formatos -->
    Tu navegador no soporta el elemento de audio ni los formatos proporcionados.
</audio>

En este ejemplo, el navegador intentará reproducir musica.ogg. Si no puede, intentará con musica.mp3. Si ninguno de los formatos es soportado o el elemento <audio> en sí no es soportado, se mostrará el mensaje de texto alternativo.
