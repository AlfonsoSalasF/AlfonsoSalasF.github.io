Clase 35: Ejercicios de HTML y Responsividad

Ejercicios Prácticos de la Clase 35

Estos ejercicios te ayudarán a aplicar los conceptos de responsividad en HTML.

Ejercicio 1: Configuración del Viewport

Modifica la estructura básica de HTML para incluir la meta etiqueta viewport esencial para el diseño responsivo. No necesitas crear un archivo CSS o JS para este ejercicio, solo la estructura HTML.

Estructura inicial:

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi Página</title>
</head>
<body>
    <h1>Hola Responsivo</h1>
    <p>Este es el contenido de la página.</p>
</body>
</html>

HTML con meta viewport:

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- ¡Aquí está! -->
    <title>Mi Página</title>
</head>
<body>
    <h1>Hola Responsivo</h1>
    <p>Este es el contenido de la página.</p>
</body>
</html>

La etiqueta <meta name="viewport" content="width=device-width, initial-scale=1.0"> es fundamental para indicar a los navegadores móviles que la página debe adaptarse al ancho del dispositivo y tener un zoom inicial de 1.0, lo que permite que el diseño responsivo funcione correctamente.

Ejercicio 2: Imágenes Responsivas con srcset

Crea una imagen que use el atributo srcset para servir diferentes versiones de la misma imagen basándose en el ancho del viewport. Usa los siguientes tamaños de imagen simulados (puedes usar via.placeholder.com):

  • imagen-pequena.jpg: 400px de ancho
  • imagen-mediana.jpg: 800px de ancho
  • imagen-grande.jpg: 1200px de ancho

Haz que la imagen ocupe:

  • 100% del ancho del viewport si el ancho es menor o igual a 600px.
  • 80% del ancho del viewport si el ancho es entre 601px y 1000px.
  • 70% del ancho del viewport si el ancho es mayor a 1000px.

Redimensiona la ventana del navegador para ver cómo la imagen cambia de tamaño (y el navegador elegiría una fuente diferente si fueran imágenes reales).

Nota: Para ver la carga de diferentes imágenes en Lighthouse, estas imágenes deberían ser diferentes archivos reales en tu servidor.

HTML con srcset y sizes:

<img srcset="https://via.placeholder.com/400x200?text=Small+400w 400w,
             https://via.placeholder.com/800x400?text=Medium+800w 800w,
             https://via.placeholder.com/1200x600?text=Large+1200w 1200w"
     sizes="(max-width: 600px) 100vw,
            (max-width: 1000px) 80vw,
            70vw"
     src="https://via.placeholder.com/1200x600?text=Default+Image"
     alt="Imagen responsiva de ejemplo">

El atributo srcset le indica al navegador las diferentes versiones de imagen disponibles con sus anchos. El atributo sizes le dice al navegador el ancho que ocupará la imagen en el layout en diferentes tamaños de pantalla (usando unidades de viewport vw). Esto permite al navegador elegir la imagen más adecuada y eficiente para la pantalla actual del usuario.

Ejercicio 3: Imágenes Responsivas con <picture>

Crea una imagen responsiva usando la etiqueta <picture>. La idea es servir diferentes formatos de imagen y/o "arte direccional" (imágenes diferentes) para distintas condiciones.

  • Si la pantalla es de al menos 800px de ancho, muestra una versión WebP horizontal.
  • Si la pantalla es de al menos 400px de ancho, muestra una versión WebP cuadrada.
  • Como fallback, usa una versión JPG pequeña.

Redimensiona la ventana del navegador. Idealmente, verías diferentes imágenes o formatos cargarse.

Nota: Para probar formatos WebP, tu navegador debe soportarlo. La URL de placeholder siempre devolverá JPG, pero simula cómo se especificarían los tipos.

HTML con <picture> y <source>:

<picture>
    <!-- Si la pantalla es de al menos 800px de ancho, muestra una versión WebP horizontal -->
    <source srcset="https://via.placeholder.com/1200x400/87CEEB/FFFFFF?text=Horizontal+WebP+Large" type="image/webp" media="(min-width: 800px)">
    <!-- Si la pantalla es de al menos 400px de ancho, muestra una versión WebP cuadrada -->
    <source srcset="https://via.placeholder.com/600x600/FFD700/000000?text=Square+WebP+Medium" type="image/webp" media="(min-width: 400px)">
    <!-- Imagen de fallback (JPG pequeña) para navegadores no compatibles o si ninguna source coincide -->
    <img src="https://via.placeholder.com/300x200/FF6347/FFFFFF?text=JPG+Small" alt="Imagen responsiva variada">
</picture>

La etiqueta <picture> permite al navegador elegir la imagen más adecuada de entre las etiquetas <source> según las condiciones de media (tamaño de pantalla) y type (formato de imagen). Si ninguna <source> coincide o el navegador no las soporta, se carga la imagen especificada en la etiqueta <img> como fallback. Esto es ideal para "arte direccional" o para servir formatos de imagen más modernos.
