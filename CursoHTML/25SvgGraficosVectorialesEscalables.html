Clase 25: SVG: Gráficos Vectoriales Escalables

En el mundo del diseño web, la calidad visual y la escalabilidad son primordiales. Los gráficos vectoriales escalables (SVG, por sus siglas en inglés) ofrecen una solución potente y flexible para integrar gráficos que se adaptan perfectamente a cualquier tamaño de pantalla o resolución sin perder calidad. En esta clase, exploraremos qué es SVG, sus ventajas y cómo puedes empezar a usarlo directamente en tu HTML.

25.1. ¿Qué es SVG y por qué usarlo?

SVG es un formato de imagen basado en XML para gráficos bidimensionales con soporte para interactividad y animación. A diferencia de los formatos de imagen rasterizados (como JPG, PNG, GIF) que se basan en píxeles, SVG describe imágenes mediante formas geométricas (círculos, rectángulos, líneas, rutas) y texto.

¿Por qué usar SVG?

  • Escalabilidad Perfecta: Los gráficos SVG no pierden calidad ni se pixelan, sin importar cuánto se escalen. Esto es crucial para el diseño web responsivo, donde los elementos gráficos deben verse bien en dispositivos con diversas resoluciones.
  • Tamaño de Archivo: A menudo, los archivos SVG son más pequeños que las imágenes rasterizadas equivalentes, especialmente para gráficos simples o logotipos.
  • Capacidad de Estilizado y Scripting: Dado que los SVG están basados en XML, pueden ser estilizados con CSS y manipulados con JavaScript, permitiendo animaciones complejas, interactividad y cambios dinámicos en el diseño.
  • Accesibilidad: El texto dentro de los SVG es seleccionable y legible por máquinas, lo que mejora la accesibilidad y el SEO.
  • Rendimiento: Al ser renderizados directamente por el navegador, los SVG pueden ser muy eficientes, especialmente para animaciones basadas en transformaciones.

25.2. Ventajas de SVG sobre imágenes rasterizadas.

Aquí hay una tabla que resume las principales ventajas de SVG frente a los formatos de imagen rasterizados:

Comparación: SVG vs. Imágenes Rasterizadas (JPG, PNG)
Característica SVG Imágenes Rasterizadas
Base Vectorial (matemáticas, formas) Rasterizada (píxeles)
Escalabilidad Escala sin pérdida de calidad (ideal para retina, pantallas de alta resolución) Se pixelan al escalar más allá de su tamaño original
Tamaño de Archivo Generalmente más pequeños para gráficos simples/vectores; más grandes para fotos complejas. Pueden ser pequeños para fotos; grandes para gráficos con muchos detalles.
Edición Fácilmente editable con editores de texto o software de diseño vectorial (Illustrator, Inkscape) Edición a nivel de píxel con software de edición de imágenes (Photoshop, GIMP)
Animación/Interactividad Totalmente animable y manipulable con CSS/JavaScript Animaciones limitadas (GIF); interactividad compleja requiere JavaScript externo.
Accesibilidad/SEO El texto es seleccionable, rastreable por motores de búsqueda. El texto forma parte de la imagen; no es seleccionable ni rastreable directamente.

25.3. Inserción de SVG directamente en HTML (<svg>).

Una de las formas más potentes de usar SVG es insertándolo directamente en tu documento HTML como código. Esto se conoce como "SVG inline" o "SVG en línea". Al hacerlo, el SVG se convierte en parte del DOM, lo que permite estilizarlo con CSS (usando selectores como svg circle { fill: red; }) y manipularlo con JavaScript.

La etiqueta principal para un gráfico SVG en línea es <svg>. Dentro de esta etiqueta, definirás las formas y elementos SVG.

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>SVG Inline Ejemplo</title>
    <style>
        svg {
            border: 1px solid #ccc;
            background-color: #f9f9f9;
        }
        .my-circle {
            fill: purple;
            stroke: black;
            stroke-width: 2;
        }
    </style>
</head>
<body>

    <h1>Mi primer SVG en línea</h1>

    <!-- El elemento SVG define un área de dibujo -->
    <svg width="200" height="150">
        <!-- Un círculo -->
        <circle cx="50" cy="50" r="40" fill="red" />

        <!-- Otro círculo con una clase CSS -->
        <circle cx="150" cy="75" r="30" class="my-circle" />
    </svg>

</body>
</html>

Al igual que con <canvas>, los atributos width y height de la etiqueta <svg> definen el tamaño del área de visualización. Sin embargo, debido a su naturaleza vectorial, el contenido dentro del SVG escalará proporcionalmente dentro de esta área.

25.4. Ejemplos básicos: <circle>, <rect>, <line>.

SVG proporciona una serie de elementos para dibujar formas básicas. Los más comunes son:

25.4.1. <circle> (Círculo)

Dibuja un círculo. Atributos importantes:

  • cx: Coordenada X del centro del círculo.
  • cy: Coordenada Y del centro del círculo.
  • r: Radio del círculo.
  • fill: Color de relleno.
  • stroke: Color del contorno.
  • stroke-width: Ancho del contorno.
<svg width="100" height="100">
    <circle cx="50" cy="50" r="40" fill="yellow" stroke="orange" stroke-width="3" />
</svg>

25.4.2. <rect> (Rectángulo)

Dibuja un rectángulo. Atributos importantes:

  • x: Coordenada X de la esquina superior izquierda.
  • y: Coordenada Y de la esquina superior izquierda.
  • width: Ancho del rectángulo.
  • height: Alto del rectángulo.
  • rx, ry (opcional): Radio de las esquinas redondeadas.
  • fill, stroke, stroke-width: (Igual que en <circle>).
<svg width="150" height="100">
    <rect x="10" y="10" width="130" height="80" fill="lightblue" stroke="darkblue" stroke-width="2" rx="10" ry="10" />
</svg>

25.4.3. <line> (Línea)

Dibuja una línea recta. Atributos importantes:

  • x1, y1: Coordenadas del punto inicial de la línea.
  • x2, y2: Coordenadas del punto final de la línea.
  • stroke: Color de la línea.
  • stroke-width: Ancho de la línea.
<svg width="150" height="100">
    <line x1="10" y1="10" x2="140" y2="90" stroke="black" stroke-width="4" />
    <line x1="140" y1="10" x2="10" y2="90" stroke="gray" stroke-width="2" />
</svg>

A diferencia de Canvas, donde dibujas píxeles de forma imperativa con JavaScript, SVG es declarativo. Defines las formas y sus propiedades directamente en el XML, y el navegador las renderiza.
