CURSO DE INICIACIÓN A HTML

Clase 22: Video: Reproducción de Video

Al igual que la etiqueta <audio>, HTML5 introdujo la etiqueta <video> para permitir la incrustación de contenido de video directamente en las páginas web sin la necesidad de plugins externos. Esto ha revolucionado la forma en que el contenido multimedia se presenta en línea, ofreciendo mayor flexibilidad, accesibilidad y compatibilidad entre dispositivos.

22.1. La etiqueta <video>.

La etiqueta <video> se utiliza para incrustar contenido de video en un documento HTML. Similar a <audio>, por sí sola no muestra controles de reproducción a menos que se especifique el atributo controls.

Ejemplo Básico:

<video src="video/ejemplo.mp4" controls width="640" height="360">
    Tu navegador no soporta el elemento de video.
</video>

El texto dentro de la etiqueta <video> (en este caso, "Tu navegador no soporta el elemento de video.") se mostrará solo si el navegador del usuario no es compatible con la etiqueta <video>.

22.2. Atributos de <video>.

La etiqueta <video> soporta varios atributos que permiten controlar el comportamiento de la reproducción, la apariencia y la carga del video.

Atributos Comunes de la Etiqueta <video>
Atributo Descripción Ejemplo
src Especifica la URL del archivo de video que se va a incrustar. Es el método más sencillo para definir la fuente. <video src="pelicula.mp4">
controls Muestra los controles de reproducción estándar del navegador (como reproducir/pausar, volumen, barra de progreso, pantalla completa). Si no se incluye, el video se reproducirá sin controles visibles a menos que se controle con JavaScript. <video src="clip.webm" controls>
autoplay Indica que el video debe comenzar a reproducirse automáticamente tan pronto como sea posible. Se debe usar con precaución, ya que puede ser intrusivo. Muchos navegadores modernos bloquean la reproducción automática con sonido a menos que el video esté silenciado. <video src="intro.mp4" autoplay>
loop Especifica que el video debe reproducirse en bucle, es decir, se reiniciará automáticamente cada vez que llegue al final. <video src="animacion.mp4" loop>
muted Indica que el video debe silenciarse por defecto al cargarse. Esto a menudo es necesario para que el atributo autoplay funcione en la mayoría de los navegadores. <video src="fondo.mp4" muted autoplay>
poster Especifica la URL de una imagen que se mostrará como un fotograma "miniatura" del video antes de que el video se cargue o se reproduzca. Es útil para dar una vista previa del contenido. <video src="viaje.mp4" poster="img/viaje_preview.jpg">
width Define el ancho del reproductor de video en píxeles. <video src="video.mp4" width="800">
height Define la altura del reproductor de video en píxeles. <video src="video.mp4" height="450">
preload Define cómo se debe cargar el archivo de video al cargar la página. Puede tener los siguientes valores:
  • none: El video no se precarga.
  • metadata: Solo se precargan los metadatos (duración, dimensiones, etc.).
  • auto: El navegador decide si precarga el video completo (valor por defecto).
 <video src="clase.mp4" preload="metadata">

Ejemplo de atributos combinados:

<video src="video/playa.mp4" controls autoplay loop muted poster="img/playa_thumb.jpg" width="720" height="405">
    Tu navegador no soporta el elemento de video.
</video>

22.3. Formatos de video compatibles.

Al igual que con el audio, no todos los navegadores soportan los mismos formatos de video. Los formatos más comunes y recomendados para la web son:

  • MP4 (MPEG-4 Part 14): Es el formato más ampliamente soportado y suele usar el códec H.264 para video y AAC para audio. Ofrece buena calidad y compresión.
  • WebM: Un formato de código abierto y libre de patentes desarrollado por Google. Utiliza el códec VP8/VP9 para video y Vorbis/Opus para audio. Es un buen complemento para MP4, especialmente en navegadores basados en Chromium y Firefox.
  • Ogg (Ogg Theora): Otro formato de código abierto. Utiliza el códec Theora para video y Vorbis para audio. Su soporte ha disminuido un poco en favor de WebM, pero aún puede ser útil para una compatibilidad más amplia en navegadores antiguos.

Para asegurar la máxima compatibilidad, es crucial proporcionar el video en múltiples formatos, lo cual se logra con la etiqueta <source>.

22.4. Múltiples fuentes con <source>.

Para ofrecer compatibilidad con diferentes navegadores y sus distintos soportes de formatos, puedes usar la etiqueta <source> dentro de <video>. El navegador intentará cargar el primer formato que reconozca y pueda reproducir.

La etiqueta <source> tiene dos atributos importantes:

  • src: La URL del archivo de video.
  • type: El tipo MIME del archivo de video (por ejemplo, "video/mp4" para MP4, "video/webm" para WebM, "video/ogg" para Ogg). Esto permite al navegador decidir si puede reproducir el archivo antes de intentar descargarlo.

Ejemplo con múltiples fuentes:

<video controls poster="img/video_preview.jpg" width="720" height="405">
    <source src="video/pelicula.webm" type="video/webm">
    <source src="video/pelicula.mp4" type="video/mp4">
    <!-- Texto alternativo si el navegador no soporta <video> o ninguno de los formatos -->
    Tu navegador no soporta el elemento de video ni los formatos proporcionados.
</video>

En este ejemplo, el navegador intentará reproducir pelicula.webm. Si no puede, intentará con pelicula.mp4. Si ninguno de los formatos es soportado o el elemento <video> en sí no es soportado, se mostrará el mensaje de texto alternativo.
