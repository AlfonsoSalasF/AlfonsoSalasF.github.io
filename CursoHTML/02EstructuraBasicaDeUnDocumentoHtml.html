CURSO DE INICIACIÓN A HTML

Clase 2: Estructura Básica de un Documento HTML.

¡Bienvenido a la segunda clase de nuestro curso de HTML! Hoy vamos a sentar las bases de todo lo que construiremos en el futuro: la estructura fundamental de un documento HTML. Comprender estos elementos básicos es crucial, ya que cada página web que crees tendrá este esqueleto. Veremos cómo se organiza un archivo HTML y la importancia de cada una de sus partes para que el navegador y los motores de búsqueda interpreten correctamente tu contenido.

2.1. La declaración <!DOCTYPE html>

Toda página HTML moderna debe comenzar con la declaración <!DOCTYPE html>. Aunque a primera vista pueda parecer una etiqueta HTML, no lo es; es una instrucción para el navegador web. Su propósito es decirle al navegador qué versión de HTML se está utilizando para renderizar la página.

En el caso de <!DOCTYPE html>, estamos indicando que el documento es una página web escrita en HTML5, la última y más extendida versión de HTML. Esta declaración es muy importante porque asegura que los navegadores rendericen la página en "modo estándar" (standards mode), lo que significa que seguirán las especificaciones CSS y HTML de manera consistente, evitando comportamientos erráticos conocidos como "modo quirks" (quirks mode).

Ejemplo de uso:

<!DOCTYPE html>
<html>
<head>
    <title>Mi primera página HTML</title>
</head>
<body>
    <p>Hola, mundo.</p>
</body>
</html>

Importancia: Sin esta declaración, los navegadores podrían intentar interpretar tu código de formas inesperadas, llevando a problemas de diseño o funcionalidad que son difíciles de depurar.

2.2. Las etiquetas <html>, <head> y <body>

Una vez que el navegador sabe que es un documento HTML5, la estructura principal se define con tres etiquetas fundamentales que encapsulan todo el contenido de tu página:

2.2.1. La Etiqueta <html> (Elemento Raíz)

La etiqueta <html> es el elemento raíz de toda página web. Contiene absolutamente todo el contenido de la página, a excepción de la declaración <!DOCTYPE html>. Es el contenedor principal que le indica al navegador dónde empieza y dónde termina el documento HTML.

Es una buena práctica incluir el atributo lang en la etiqueta <html> para especificar el idioma principal del contenido de la página (por ejemplo, lang="es" para español). Esto ayuda a los motores de búsqueda, a los lectores de pantalla y a los navegadores a presentar el contenido de manera más adecuada.

Ejemplo:

<!DOCTYPE html>
<html lang="es">
    <!-- Aquí irá todo el contenido de la página -->
</html>

2.2.2. La Etiqueta <head> (Metadatos)

La etiqueta <head> es un contenedor para los metadatos de la página HTML. Los metadatos son información sobre la página, pero que no se muestra directamente en el navegador al usuario. Incluye elementos como el título de la página (que aparece en la pestaña del navegador), enlaces a hojas de estilo CSS, scripts JavaScript, información sobre el autor, descripción para motores de búsqueda, etc.

Es crucial para la SEO (Optimización para Motores de Búsqueda) y para la forma en que el navegador maneja la página.

Ejemplo:

<head>
    <meta charset="UTF-8">
    <title>Mi Página Asombrosa</title>
    <link rel="stylesheet" href="estilos.css">
</head>

2.2.3. La Etiqueta <body> (Contenido Visible)

La etiqueta <body> es donde resides todo el contenido visible de tu página web. Todo lo que el usuario ve y con lo que interactúa (texto, imágenes, videos, enlaces, botones, formularios, etc.) se coloca dentro de esta etiqueta.

Es el "cuerpo" de tu documento y el área donde la mayor parte de tu trabajo de construcción de la página se llevará a cabo.

Ejemplo:

<body>
    <h1>Bienvenido a mi sitio web</h1>
    <p>Este es el contenido principal de mi página.</p>
</body>

2.3. Metadatos Importantes en <head>

Dentro de la etiqueta <head>, hay varios elementos <meta> y <title> que son especialmente importantes:

2.3.1. La Etiqueta <title>

La etiqueta <title> define el título de la página web. Este título aparece en la barra de título del navegador o en la pestaña de la ventana. Es fundamental para la usabilidad y para la SEO.

  • Es lo primero que los usuarios ven en los resultados de búsqueda.
  • Es el nombre que se usa cuando los usuarios guardan tu página en favoritos o marcadores.
  • Debe ser conciso, descriptivo y relevante para el contenido de la página.

Ejemplo:

<head>
    <title>Página Principal - Mi Blog de Viajes</title>
</head>

2.3.2. <meta charset="UTF-8">

Esta etiqueta <meta> es de suma importancia. Especifica la codificación de caracteres del documento. "UTF-8" es la codificación de caracteres universalmente recomendada y más utilizada. Asegura que todos los caracteres, incluyendo tildes, eñes y otros caracteres especiales de diferentes idiomas, se muestren correctamente en el navegador, evitando problemas de visualización (los famosos "caracteres extraños" o "mojibake").

Debe ser el primer elemento dentro de <head>, justo después de <head>, para que el navegador sepa cómo interpretar el resto del documento desde el principio.

Ejemplo:

<head>
    <meta charset="UTF-8">
    <title>Mi Página con UTF-8</title>
</head>

2.3.3. <meta name="viewport" content="width=device-width, initial-scale=1.0">

Esta es otra etiqueta <meta> crucial para el diseño web responsivo, que asegura que tu página se vea bien en diferentes dispositivos (ordenadores de escritorio, tabletas, teléfonos móviles).

  • width=device-width: Establece el ancho del viewport (la ventana visible del navegador) al ancho del dispositivo.
  • initial-scale=1.0: Establece el nivel de zoom inicial cuando la página se carga por primera vez.

Sin esta etiqueta, los dispositivos móviles a menudo intentarían renderizar la página como si fuera una pantalla de escritorio grande y luego la encogerían, lo que resultaría en texto pequeño y elementos difíciles de tocar.

Ejemplo:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Página Adaptable</title>
</head>

2.4. Guardar y Abrir un Archivo HTML en el Navegador

Una vez que hayas escrito tu código HTML, el siguiente paso es verlo en acción en un navegador web. El proceso es sencillo:

  1. Guardar el archivo:
    • Abre tu editor de texto (como VS Code, Sublime Text, Notepad++).
    • Crea un nuevo archivo y pega tu código HTML.
    • Guarda el archivo con la extensión .html o .htm (.html es más común). Por ejemplo: mi_primera_pagina.html o index.html (index.html es el nombre por defecto para la página principal de un sitio web).
    • Elige una ubicación en tu ordenador donde puedas encontrarlo fácilmente.
  2. Abrir en el navegador:
    • Navega hasta la ubicación donde guardaste el archivo.
    • Haz doble clic en el archivo .html. Tu sistema operativo lo abrirá automáticamente con tu navegador web predeterminado (Chrome, Firefox, Edge, Safari, etc.).
    • Alternativamente, puedes arrastrar y soltar el archivo .html directamente en la ventana de tu navegador web.
    • También puedes hacer clic derecho sobre el archivo y seleccionar "Abrir con..." y elegir tu navegador preferido.

¡Felicidades! Ahora estás viendo tu primera página web en un navegador. Cada vez que realices cambios en tu archivo HTML, deberás guardar el archivo en tu editor y luego refrescar la página en el navegador (generalmente presionando F5 o el icono de recargar) para ver los cambios.
