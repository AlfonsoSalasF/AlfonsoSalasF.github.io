CURSO DE INICIACIÓN A HTML

Clase 35: HTML y Responsividad (Introducción)

En el mundo actual, los usuarios acceden a la web desde una vasta gama de dispositivos: ordenadores de escritorio, portátiles, tabletas, smartphones, e incluso smart TVs. Un diseño web que se vea bien y funcione de manera óptima en todas estas pantallas es fundamental. Aquí es donde entra en juego el "diseño web responsivo" o "responsive web design".

35.1. Concepto de diseño web responsivo.

El diseño web responsivo es una aproximación al diseño y desarrollo web que busca crear sitios que proporcionen una experiencia de visualización y uso óptima en una amplia gama de dispositivos (desde monitores de escritorio grandes hasta teléfonos móviles). Un sitio web responsivo se "adapta" o "responde" al tamaño de la pantalla del usuario, ajustando automáticamente su diseño, imágenes y contenido para que se vean bien y sean fáciles de usar.

Principios clave del diseño responsivo:

  • Grid flexibles: El diseño no se basa en tamaños fijos de píxeles, sino en porcentajes y unidades relativas para que los elementos se redimensionen proporcionalmente.
  • Imágenes flexibles: Las imágenes se escalan para encajar en el layout, evitando desbordamientos o tamaños excesivos.
  • Media queries de CSS: Son reglas de CSS que permiten aplicar estilos diferentes en función de características del dispositivo como el ancho de la pantalla, la altura, la orientación, la resolución, etc. Por ejemplo, puedes ocultar un menú de navegación lateral en pantallas pequeñas y mostrar un icono de hamburguesa.
  • Prioridad de contenido: En pantallas pequeñas, el contenido se reorganiza, y a veces se oculta contenido menos importante, para asegurar que la información clave sea accesible y legible.

¿Por qué es importante?

  • Experiencia de usuario: Mejora la usabilidad y la satisfacción del usuario al no tener que hacer zoom o desplazarse excesivamente.
  • SEO: Google y otros motores de búsqueda priorizan los sitios web "mobile-friendly" en sus resultados de búsqueda móvil.
  • Futuro a prueba: Un diseño responsivo es más adaptable a nuevos tamaños de pantalla y dispositivos que puedan surgir.

35.2. Meta viewport: <meta name="viewport" content="width=device-width, initial-scale=1.0">.

El meta tag viewport es una pieza fundamental para el diseño web responsivo en HTML. Sin él, los navegadores móviles intentarían renderizar la página como si fuera un escritorio, luego la reducirían para ajustarla a la pantalla pequeña, lo que resultaría en texto e imágenes diminutos y difíciles de leer.

Explicación del atributo:

<meta name="viewport" content="width=device-width, initial-scale=1.0">
  • name="viewport": Indica que esta meta etiqueta está relacionada con el viewport del navegador.
  • content="": Contiene las configuraciones para el viewport.
    • width=device-width: Establece el ancho del viewport al ancho del dispositivo en píxeles CSS. Esto asegura que la página ocupe todo el ancho disponible de la pantalla, sin importar su tamaño físico.
    • initial-scale=1.0: Establece el nivel de zoom inicial cuando la página se carga por primera vez. Un valor de 1.0 significa que no hay zoom y que 1 píxel CSS es igual a 1 píxel de dispositivo, lo que es óptimo para la mayoría de los diseños responsivos.

Ubicación: Esta etiqueta siempre debe ir dentro de la sección <head> de tu documento HTML.

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Página Responsiva</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    ...
</body>
</html>

¡Crucial! No incluir el meta viewport, o configurarlo incorrectamente, es uno de los errores más comunes que causan que una página no sea "mobile-friendly". Siempre inclúyelo en todos tus documentos HTML.

35.3. Imágenes responsivas: <picture> y srcset.

Las imágenes pueden ser un desafío en el diseño responsivo, ya que una imagen de alta resolución diseñada para un monitor de escritorio puede ser innecesariamente grande y lenta de cargar en un dispositivo móvil. HTML5 introduce soluciones para servir imágenes optimizadas para diferentes tamaños de pantalla y densidades de píxeles.

35.3.1. El atributo srcset para <img>.

El atributo srcset permite especificar una lista de fuentes de imagen diferentes junto con descriptores que le indican al navegador cuándo usar cada una. El navegador elegirá la imagen más adecuada basándose en el ancho del viewport, la densidad de píxeles de la pantalla y la densidad de píxeles del dispositivo.

  • Descriptores de ancho (w): Indica el ancho intrínseco de cada imagen. El navegador elegirá la imagen que mejor se ajuste al ancho del espacio que ocupará la imagen en el layout.
  • Descriptores de densidad de píxeles (x): Indica la densidad de píxeles de la imagen (ej., 1x para pantallas estándar, 2x para pantallas Retina).
<!-- Ejemplo con descriptores de ancho (w) -->
<img srcset="imagen-pequena.jpg 480w,
             imagen-mediana.jpg 800w,
             imagen-grande.jpg 1200w"
     sizes="(max-width: 600px) 480px,
            (max-width: 1000px) 800px,
            1200px"
     src="imagen-grande.jpg" alt="Descripción de la imagen">

<!-- Ejemplo con descriptores de densidad de píxeles (x) -->
<img srcset="imagen-1x.png 1x,
             imagen-2x.png 2x"
     src="imagen-1x.png" alt="Logotipo">

El atributo sizes (usado con w) es crucial. Le dice al navegador qué tan ancha será la imagen en diferentes puntos de interrupción de CSS, lo que le ayuda a seleccionar la imagen correcta de srcset antes de que se cargue el CSS.

35.3.2. La etiqueta <picture>.

La etiqueta <picture> ofrece aún más control sobre las imágenes responsivas, permitiéndote especificar diferentes imágenes para diferentes condiciones, como formatos de imagen (WebP, JPEG), o arte direccional (recortes diferentes de la misma imagen).

Usa la etiqueta <source> dentro de <picture> con el atributo media (para media queries) o type (para formatos MIME). El navegador elegirá el primer <source> cuyo media query sea verdadero y cuyo type sea compatible. El <img> final es el "fallback" o la imagen por defecto si ningún <source> coincide o es compatible.

<picture>
    <!-- Sirve una imagen WebP para pantallas grandes -->
    <source srcset="imagen-grande.webp" type="image/webp" media="(min-width: 1200px)">
    <!-- Sirve una imagen WebP para pantallas medianas -->
    <source srcset="imagen-mediana.webp" type="image/webp" media="(min-width: 768px)">
    <!-- Sirve una imagen JPEG para pantallas pequeñas (o si WebP no es compatible) -->
    <source srcset="imagen-pequena.jpg" type="image/jpeg">
    <!-- Imagen de fallback para navegadores no compatibles con <picture> o si ninguna source coincide -->
    <img src="imagen-pequena.jpg" alt="Descripción de la imagen responsiva">
</picture>

Recuerda: La optimización de imágenes es una parte crucial del rendimiento web. Usar srcset y <picture> junto con el meta viewport te ayudará a ofrecer una experiencia más rápida y adaptada a tus usuarios.
