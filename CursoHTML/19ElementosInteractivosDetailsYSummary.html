CURSO DE INICIACIÓN A HTML

Clase 19: Elementos Interactivos: <details> y <summary>.

HTML5 introdujo elementos que nos permiten crear componentes de interfaz de usuario interactivos de forma nativa, sin necesidad de JavaScript. Entre ellos, <details> y <summary> son perfectos para implementar secciones de contenido expandibles y colapsables, como preguntas frecuentes (FAQ) o detalles adicionales.

19.1. Creación de secciones expandibles/colapsables de contenido con <details> y <summary>.

El elemento <details> crea un widget de divulgación a través del cual el usuario puede obtener información adicional o controles. El elemento <summary> proporciona un resumen o leyenda visible para el <details>, que sirve como un control para alternar la visibilidad del resto del contenido del <details>.

Cuando el usuario hace clic en el <summary>, el contenido dentro de <details> se expande o se contrae. Por defecto, el contenido está colapsado.

Estructura básica:

<details>
    <summary>Título o pregunta visible</summary>
    <p>Contenido detallado que se expande/contrae.</p>
    <ul>
        <li>Elemento 1</li>
        <li>Elemento 2</li>
    </ul>
</details>

19.1.1. Uso de <details> y <summary>.

El <summary> debe ser el primer hijo del elemento <details>. Todo el contenido que sigue a <summary> dentro de <details> será el contenido que se mostrará u ocultará.

Ejemplo: Pregunta Frecuente (FAQ).

<div>
    <details>
        <summary>¿Qué es HTML?</summary>
        <p>HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web y aplicaciones web. Se utiliza para estructurar el contenido de una página, como encabezados, párrafos, imágenes y enlaces.</p>
    </details>
    <br>
    <details>
        <summary>¿Cuál es la diferencia entre HTML y CSS?</summary>
        <p>HTML estructura el contenido de una página web, mientras que CSS (Cascading Style Sheets) se utiliza para estilizar ese contenido, controlando la presentación visual como colores, fuentes, espaciado y diseño.</p>
    </details>
</div>
¿Qué es HTML?

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web y aplicaciones web. Se utiliza para estructurar el contenido de una página, como encabezados, párrafos, imágenes y enlaces.


¿Cuál es la diferencia entre HTML y CSS?

HTML estructura el contenido de una página web, mientras que CSS (Cascading Style Sheets) se utiliza para estilizar ese contenido, controlando la presentación visual como colores, fuentes, espaciado y diseño.

19.2. Atributo open.

El atributo booleano open en la etiqueta <details> indica si los detalles de contenido deben ser visibles para el usuario al cargar la página. Si el atributo open está presente, el contenido estará expandido por defecto. Si está ausente, el contenido estará colapsado por defecto.

Ejemplo: Sección de "Avisos importantes" abierta por defecto.

<details open>
    <summary>Avisos Importantes</summary>
    <ul>
        <li>Mantenimiento programado para el próximo viernes a las 02:00 AM.</li>
        <li>Nuevas funcionalidades añadidas a su panel de control.</li>
    </ul>
</details>
Avisos Importantes
  • Mantenimiento programado para el próximo viernes a las 02:00 AM.
  • Nuevas funcionalidades añadidas a su panel de control.

Ejemplo: Sección colapsada por defecto.

<details>
    <summary>Ver detalles de contacto</summary>
    <p>Teléfono: +34 987 654 321</p>
    <p>Email: info@ejemplo.com</p>
</details>
Ver detalles de contacto

Teléfono: +34 987 654 321

Email: info@ejemplo.com

19.3. Casos de uso para <details> y <summary>.

Estos elementos son muy versátiles y pueden mejorar la usabilidad y la organización del contenido en diversas situaciones:

  • Preguntas Frecuentes (FAQ): Es el uso más común, permitiendo mostrar solo las preguntas y expandir las respuestas al hacer clic.
  • Detalles Opcionales: Información adicional que no es crucial para todos los usuarios, pero que está disponible si se desea.
  • Secciones Colapsables: En paneles de administración, filtros de búsqueda avanzados, o cualquier lugar donde quieras ocultar/mostrar un bloque de contenido para ahorrar espacio o reducir la sobrecarga visual.
  • Acordeones Simples: Aunque para acordeones complejos con interacciones avanzadas podrías necesitar JavaScript, para los básicos, <details> es una solución nativa y accesible.

Accesibilidad integrada: Una de las mayores ventajas de <details> y <summary> es que gestionan automáticamente la semántica de accesibilidad (roles ARIA, estados) sin que el desarrollador tenga que añadir atributos específicos.
